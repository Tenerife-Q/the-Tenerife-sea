88# 链地址法哈希表 - 交互式程序

## 介绍

这是一个演示链地址法解决哈希冲突的交互式程序。它与线性探测和二次探测版本有相同的操作界面和功能，方便比较不同哈希冲突解决方案的性能。

## 主要特性

1. **链地址法冲突解决**：使用链表处理哈希地址冲突
2. **完整交互式界面**：与线性/二次探测版本一致的菜单系统
3. **冲突可视化**：直观显示冲突处理过程
4. **性能统计**：自动计算平均查找长度(ASL)
5. **多种测试模式**：包含集中冲突、分散冲突和混合模式测试

## 使用方法

### 运行程序

使用批处理文件启动：
```
run_chain_demo.bat     # 中文版
run_chain_demo_en.bat  # 英文版
```

或手动编译运行：
```
gcc -o interactive_chain_division.exe interactive_chain_division.c
interactive_chain_division.exe
```

```
5. 🔄 重置哈希表 - 清空哈希表
6. 🧪 随机冲突测试 - 测试不同冲突模式

```

## 链地址法特点

与线性探测和二次探测相比，链地址法有以下特点：

1. **无开放定址**：不需要在表中寻找其他位置，而是使用链表存储冲突元素
2. **无聚集现象**：不会形成元素聚集区域
3. **装载因子可大于1**：可以存储比表大小更多的元素
4. **ASL特性**：成功查找的平均长度一般小于开放定址法

## 测试建议

1. 运行**选项6**：观察不同类型的冲突如何形成链表
2. 运行**选项7**：观察随机数据的表现
3. **对比实验**：使用相同数据测试链地址法和线性/二次探测法
4. **ASL比较**：观察不同方法在高装载因子下的性能差异

## 理论分析

链地址法的ASL理论值：
- 成功查找：约为 1 + α/2 (α为装载因子)
- 不成功查找：约为 α (α为装载因子)

这与开放定址法的理论值有显著不同，尤其是在高装载因子时。

## 适用场景

链地址法特别适合：
- 装载因子较高的情况
- 无法预估元素数量的场景
- 需要快速删除操作的应用
