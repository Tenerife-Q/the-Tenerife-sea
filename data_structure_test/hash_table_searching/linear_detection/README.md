# 哈希表程序综合文档

## 程序功能概述
这是一个基于除留余数法和线性探测的交互式哈希表演示程序，支持插入、查找、删除、重置、ASL统计、冲突可视化、批量测试（集中/分散/混合/伪随机），并自动生成日志记录不同测试集的实验情况。

## 快速入门指南

```bash
双击 demo_conflicts.bat
```

```bash
.\interactive_hash_table.exe
```

## 冲突模式分析

### 集中冲突测试数据
```
键值: 13, 26, 39, 52, 65
哈希映射: 
- 13 % 13 = 0 → 位置 0
- 26 % 13 = 0 → 位置 0 (冲突) → 位置 1
- 39 % 13 = 0 → 位置 0 (冲突) → 位置 1 (冲突) → 位置 2
- 52 % 13 = 0 → 位置 0,1,2 (冲突) → 位置 3
- 65 % 13 = 0 → 位置 0,1,2,3 (冲突) → 位置 4
```
**特点**: 形成长度为5的连续冲突链，ASL = (1+2+3+4+5)/5 = 3.0

### 分散冲突测试数据
```
键值: 14, 27, 41, 29, 16, 56, 69
哈希映射:
- 14 % 13 = 1 → 位置 1
- 27 % 13 = 1 → 位置 1 (冲突) → 位置 2
- 41 % 13 = 2 → 位置 2 (冲突) → 位置 3
- 29 % 13 = 3 → 位置 3 (冲突) → 位置 4
- 16 % 13 = 3 → 位置 3,4 (冲突) → 位置 5
- 56 % 13 = 4 → 位置 4,5 (冲突) → 位置 6
- 69 % 13 = 4 → 位置 4,5,6 (冲突) → 位置 7
```
**特点**: 形成多个短冲突链，分布较均匀

### 混合冲突测试数据
```
键值: 15, 28, 33, 46, 51, 64, 77, 82, 95
哈希映射分析:
- 15 % 13 = 2
- 28 % 13 = 2 (与15冲突)
- 33 % 13 = 7
- 46 % 13 = 7 (与33冲突)
- 51 % 13 = 12
- 64 % 13 = 12 (与51冲突)
- 77 % 13 = 12 (与51,64冲突)
- 82 % 13 = 4
- 95 % 13 = 4 (与82冲突)
```
**特点**: 模拟真实场景，既有2元素冲突，也有3元素冲突

## 测试数据改进分析

### 原始测试数据问题分析
#### 旧数据集 {13, 26, 39, 21, 34, 45, 56, 67, 78}
```
哈希映射 (key % 13):
13 % 13 = 0 → 位置 0
26 % 13 = 0 → 位置 0 (冲突) → 位置 1  
39 % 13 = 0 → 位置 0,1 (冲突) → 位置 2
21 % 13 = 8 → 位置 8
34 % 13 = 8 → 位置 8 (冲突) → 位置 9
45 % 13 = 6 → 位置 6
56 % 13 = 4 → 位置 4
67 % 13 = 2 → 位置 2 (冲突) → 位置 3
78 % 13 = 0 → 位置 0,1,2,3 (冲突) → 位置 4 (冲突) → 位置 5
```
**问题**:
- 过度集中在位置0，形成长冲突链 (0→1→2→3→4→5)
- 分布不均匀，某些区域密集，某些区域空闲
- 不能很好展示线性探测的各种情况

### 新测试数据优化设计
#### 新数据集 {17, 30, 22, 48, 35, 61, 44, 19, 57}
```
哈希映射 (key % 13):
17 % 13 = 4  → 位置 4
30 % 13 = 4  → 位置 4 (冲突) → 位置 5
22 % 13 = 9  → 位置 9  
48 % 13 = 9  → 位置 9 (冲突) → 位置 0
35 % 13 = 9  → 位置 9,0 (冲突) → 位置 1
61 % 13 = 9  → 位置 9,0,1 (冲突) → 位置 2
44 % 13 = 5  → 位置 5 (冲突) → 位置 6
19 % 13 = 6  → 位置 6 (冲突) → 位置 7
57 % 13 = 5  → 位置 5,6,7 (冲突) → 位置 8
```
**优势**:
- 形成多个分离的冲突链：
  - 链1: 位置4→5
  - 链2: 位置9→0→1→2 (环形)
  - 链3: 位置5→6→7→8
- 展示了环形探测（从9到0）
- 冲突分布更均匀，更接近实际应用
