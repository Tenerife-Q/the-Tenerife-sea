# 交互式哈希表程序使用说明

## 程序功能
这是一个基于除留余数法和线性探测的交互式哈希表演示程序。

## 主要特性
1. **除留余数法哈希函数**: 使用 `key % 13` 计算初始哈希位置
2. **线性探测冲突解决**: 当发生冲突时，依次检查下一个位置
3. **完整的CRUD操作**: 插入、查找、删除、显示
4. **冲突可视化**: 清晰显示冲突发生和解决过程
5. **ASL计算**: 实时显示平均查找长度(Average Search Length)
6. **表格化显示**: 美观的表格展示哈希表状态

## 菜单选项
```
1. 📥 插入元素 - 向哈希表中插入新的键值
2. 🔍 查找元素 - 在哈希表中查找指定键值
3. 🗑️ 删除元素 - 从哈希表中删除指定键值
4. 📊 显示哈希表 - 显示当前哈希表的完整状态
5. 🔄 重置哈希表 - 清空哈希表，重新开始
6. 🧪 随机冲突测试 - 自动测试不同冲突模式
7. 🎲 伪随机数测试 - 使用伪随机数据测试
8. ❌ 退出程序 - 退出程序
```

## 新增测试功能

### 🧪 随机冲突测试 (选项6)
包含三个测试集，展示不同的冲突模式：

**测试集1：集中冲突**
- 键值: 13, 26, 39, 52, 65
- 特点: 所有键值都映射到位置0，形成长冲突链
- 目的: 观察最坏情况下的性能表现

**测试集2：分散冲突**  
- 键值: 14, 27, 41, 29, 16, 56, 69
- 特点: 键值分散到不同初始位置，形成多个短冲突链
- 目的: 观察较好分布下的性能表现

**测试集3：混合冲突**
- 键值: 15, 28, 33, 46, 51, 64, 77, 82, 95
- 特点: 既有连续冲突，也有分散冲突
- 目的: 模拟真实使用场景

### 🎲 伪随机数测试 (选项7)
- 使用线性同余生成器产生伪随机键值
- 生成10个随机数进行插入测试
- 随后进行随机查找测试
- 目的: 测试哈希表在随机数据下的表现

## 使用示例

### 启动程序
```bash
.\interactive_hash_table.exe
```

### 示例操作序列
1. 输入哈希表大小：`13`
2. 选择随机冲突测试：`6` (观察不同冲突模式)
3. 选择伪随机测试：`7` (测试随机数据)
4. 手动插入测试：`1`, 输入`13` (观察冲突处理)
5. 继续插入：`26`, `39` (观察冲突链)
6. 显示哈希表：`4` (查看当前状态和ASL)
7. 查找元素：`2`, 输入`26` (显示查找过程)
8. 删除元素：`3`, 输入`13`
9. 重新显示：`4` (观察删除后的状态变化)

## 冲突演示
当插入以下键值时，会发生有趣的冲突：
- `13 % 13 = 0` (位置0)
- `26 % 13 = 0` (位置0冲突，探测到位置1)
- `39 % 13 = 0` (位置0、1冲突，探测到位置2)

## ASL计算说明
平均查找长度 = 所有已存储元素的探测次数总和 / 已存储元素数量

例如：
- 元素13在位置0，探测1次
- 元素26在位置1，探测2次  
- 元素39在位置2，探测3次
- ASL = (1+2+3)/3 = 2.00

.\interactive_hash_table.exe

## 技术特点
- **状态管理**: 0=空闲, 1=已占用, 2=已删除
- **内存管理**: 自动管理内存分配和释放
- **错误处理**: 完善的输入验证和错误提示
- **用户友好**: 直观的UI和清晰的操作提示

## 测试建议
1. **基本操作测试**：插入、查找、删除
2. **自动冲突测试**：使用选项6观察三种不同冲突模式
3. **随机数据测试**：使用选项7测试伪随机数据
4. **边界情况测试**：满表、空表、重复插入
5. **ASL性能分析**：在不同装载因子和冲突模式下观察ASL变化
6. **冲突链对比**：比较集中冲突vs分散冲突的性能差异

## 性能观察要点
- **集中冲突**: ASL较高，长冲突链影响性能
- **分散冲突**: ASL较低，多个短链分布均匀  
- **混合模式**: 接近实际使用场景的表现
- **随机数据**: 最接近理论期望的性能表现
