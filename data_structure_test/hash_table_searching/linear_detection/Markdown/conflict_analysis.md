# 哈希表冲突模式分析报告

## 测试数据设计

### 1. 集中冲突测试数据
```
键值: 13, 26, 39, 52, 65
哈希映射: 
- 13 % 13 = 0 → 位置 0
- 26 % 13 = 0 → 位置 0 (冲突) → 位置 1
- 39 % 13 = 0 → 位置 0 (冲突) → 位置 1 (冲突) → 位置 2
- 52 % 13 = 0 → 位置 0,1,2 (冲突) → 位置 3
- 65 % 13 = 0 → 位置 0,1,2,3 (冲突) → 位置 4
```
**特点**: 形成长度为5的连续冲突链，ASL = (1+2+3+4+5)/5 = 3.0

### 2. 分散冲突测试数据
```
键值: 14, 27, 41, 29, 16, 56, 69
哈希映射:
- 14 % 13 = 1 → 位置 1
- 27 % 13 = 1 → 位置 1 (冲突) → 位置 2
- 41 % 13 = 2 → 位置 2 (冲突) → 位置 3
- 29 % 13 = 3 → 位置 3 (冲突) → 位置 4
- 16 % 13 = 3 → 位置 3,4 (冲突) → 位置 5
- 56 % 13 = 4 → 位置 4,5 (冲突) → 位置 6
- 69 % 13 = 4 → 位置 4,5,6 (冲突) → 位置 7
```
**特点**: 形成多个短冲突链，分布较均匀

### 3. 混合冲突测试数据
```
键值: 15, 28, 33, 46, 51, 64, 77, 82, 95
哈希映射分析:
- 15 % 13 = 2
- 28 % 13 = 2 (与15冲突)
- 33 % 13 = 7
- 46 % 13 = 7 (与33冲突)
- 51 % 13 = 12
- 64 % 13 = 12 (与51冲突)
- 77 % 13 = 12 (与51,64冲突)
- 82 % 13 = 4
- 95 % 13 = 4 (与82冲突)
```
**特点**: 模拟真实场景，既有2元素冲突，也有3元素冲突

## 性能对比分析

| 测试类型 | 冲突特征 | 预期ASL | 实际表现 | 适用场景 |
|----------|----------|---------|----------|----------|
| 集中冲突 | 单一长链 | 3.0 | 最差 | 最坏情况分析 |
| 分散冲突 | 多个短链 | ~1.5 | 较好 | 良好哈希函数 |
| 混合冲突 | 中等长度 | ~2.0 | 中等 | 实际使用场景 |
| 伪随机 | 随机分布 | ~1.3 | 最好 | 理想情况 |

## 线性探测行为观察

### 优点
1. **实现简单**: 探测序列为 h(k), h(k)+1, h(k)+2, ...
2. **缓存友好**: 连续内存访问
3. **空间效率**: 不需要额外存储空间

### 缺点  
1. **聚集现象**: 连续占用的槽位会形成"聚集"
2. **性能退化**: 聚集越严重，查找时间越长
3. **删除复杂**: 需要标记删除而非直接删除

## 实验建议

### 测试流程
1. **启动程序**，设置表大小为15（便于观察）
2. **执行选项6**：观察三种冲突模式的差异
3. **执行选项7**：测试随机数据性能
4. **手动测试**：插入特定数据验证理论分析

### 观察要点
- 每次插入后的探测次数
- ASL的变化趋势
- 冲突链的形成模式
- 不同装载因子下的性能差异

## 理论公式验证

### 成功查找的期望探测次数
```
E[成功查找] ≈ 1 + α/2
其中 α = n/m (装载因子)
```

### 不成功查找的期望探测次数
```
E[不成功查找] ≈ 1 + α²/2
```

通过实际测试验证这些理论公式的准确性。
