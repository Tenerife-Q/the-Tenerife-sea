# 快速入门指南

## 🚀 立即开始

### 方法1：一键启动
```bash
双击 demo_conflicts.bat
```

### 方法2：手动运行
```bash
gcc -o interactive_hash_table.exe interactive_hash_table.c
.\interactive_hash_table.exe
```

## 📋 推荐测试序列

### 第一次使用 - 基础体验
1. 输入哈希表大小：`15`
2. 选择 `6` - 随机冲突测试
3. 观察三种不同的冲突模式
4. 选择 `4` - 查看最终状态

### 第二次使用 - 深入理解  
1. 重置表：选择 `5`
2. 选择 `7` - 伪随机数测试
3. 观察随机数据的性能表现
4. 选择 `4` - 对比ASL差异

### 第三次使用 - 手动验证
1. 重置表：选择 `5`
2. 手动插入：`13, 26, 39` (观察连续冲突)
3. 查找测试：搜索 `26` (观察探测过程)
4. 删除测试：删除 `13` (观察状态变化)

## 🎯 关键观察点

### 冲突模式对比
- **集中冲突**: 长冲突链，ASL较高
- **分散冲突**: 短冲突链，ASL较低  
- **混合模式**: 接近实际场景
- **随机数据**: 最优性能表现

### ASL变化规律
```
装载因子 = 已存储元素 / 表大小
ASL ≈ 1 + 装载因子/2 (理论公式)
```

### 探测过程可视化
程序会详细显示：
- 初始哈希位置计算
- 每次冲突的处理过程
- 最终存储位置
- 总探测次数

## 📊 性能评估

观察不同情况下的ASL值：
- **优秀**: ASL < 1.5
- **良好**: 1.5 ≤ ASL < 2.5  
- **一般**: 2.5 ≤ ASL < 3.5
- **较差**: ASL ≥ 3.5

## 💡 实验建议

1. **对比实验**: 在相同装载因子下比较不同冲突模式
2. **装载因子实验**: 观察ASL随装载因子的变化
3. **边界测试**: 测试表满时的行为
4. **删除影响**: 观察删除操作对性能的影响

## 🔧 故障排除

**如果程序无法启动**:
- 确保已安装GCC编译器
- 检查文件路径是否正确
- 尝试手动编译：`gcc -o interactive_hash_table.exe interactive_hash_table.c`

**如果显示乱码**:
- 在Windows命令行中运行
- 或使用 `chcp 65001` 设置UTF-8编码

开始探索哈希表的奥秘吧！🎉
