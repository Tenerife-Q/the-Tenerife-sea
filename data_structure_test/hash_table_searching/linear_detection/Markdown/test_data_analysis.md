# 测试数据改进分析

## 原始测试数据问题分析

### 旧数据集 {13, 26, 39, 21, 34, 45, 56, 67, 78}
```
哈希映射 (key % 13):
13 % 13 = 0 → 位置 0
26 % 13 = 0 → 位置 0 (冲突) → 位置 1  
39 % 13 = 0 → 位置 0,1 (冲突) → 位置 2
21 % 13 = 8 → 位置 8
34 % 13 = 8 → 位置 8 (冲突) → 位置 9
45 % 13 = 6 → 位置 6
56 % 13 = 4 → 位置 4
67 % 13 = 2 → 位置 2 (冲突) → 位置 3
78 % 13 = 0 → 位置 0,1,2,3 (冲突) → 位置 4 (冲突) → 位置 5
```

**问题**:
- 过度集中在位置0，形成长冲突链 (0→1→2→3→4→5)
- 分布不均匀，某些区域密集，某些区域空闲
- 不能很好展示线性探测的各种情况

## 新测试数据优化设计

### 新数据集 {17, 30, 22, 48, 35, 61, 44, 19, 57}
```
哈希映射 (key % 13):
17 % 13 = 4  → 位置 4
30 % 13 = 4  → 位置 4 (冲突) → 位置 5
22 % 13 = 9  → 位置 9  
48 % 13 = 9  → 位置 9 (冲突) → 位置 0
35 % 13 = 9  → 位置 9,0 (冲突) → 位置 1
61 % 13 = 9  → 位置 9,0,1 (冲突) → 位置 2
44 % 13 = 5  → 位置 5 (冲突) → 位置 6
19 % 13 = 6  → 位置 6 (冲突) → 位置 7
57 % 13 = 5  → 位置 5,6,7 (冲突) → 位置 8
```

**优势**:
- 形成多个分离的冲突链：
  - 链1: 位置4→5
  - 链2: 位置9→0→1→2 (环形)
  - 链3: 位置5→6→7→8
- 展示了环形探测（从9到0）
- 冲突分布更均匀，更接近实际应用

## 冲突模式对比

| 方面 | 旧数据集 | 新数据集 |
|------|----------|----------|
| 冲突链数量 | 3个 | 3个 |
| 最长冲突链 | 6个元素 | 4个元素 |
| 分布均匀性 | 差 | 好 |
| 环形探测 | 无 | 有 |
| 实际场景模拟 | 差 | 好 |
| 预期ASL | 3.2 | 2.4 |

## 性能指标预期

### 理论ASL计算
装载因子 α = 9/10 = 0.9

**旧数据集ASL**:
(1+2+3+1+2+1+1+2+3)/9 ≈ 1.78

**新数据集ASL**:
(1+2+1+2+3+4+3+4+5)/9 ≈ 2.78

### 查找效率分析
新数据集虽然ASL略高，但：
- 冲突分布更均匀
- 更好地展示线性探测特性
- 包含环形探测场景
- 更接近随机数据的表现

## 教学价值提升

### 展示的概念
1. **环形探测**: 48从位置9探测到位置0
2. **多点冲突**: 不只是单一位置的冲突
3. **分散聚集**: 冲突分布在表的不同区域
4. **实际性能**: 更接近真实应用场景

### 学习效果
- 更好理解哈希表的动态行为
- 观察不同冲突模式的影响
- 体验线性探测的完整流程
- 分析ASL与数据分布的关系

这种改进让测试数据更具代表性和教学价值！
