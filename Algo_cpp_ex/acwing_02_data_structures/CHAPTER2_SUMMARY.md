## 📝 学习笔记

### 核心思想
数据结构的核心在于**如何高效地组织数据**以支持特定的操作（增删改查）。在算法竞赛中，通常使用**静态数组模拟**链表、树、堆等结构，以避免 `new/delete` 的开销并获得更好的缓存命中率。

### 常见陷阱
1. **下标问题**：
   - 单链表头结点 `head` 初始为 -1。
   - 双链表 `idx` 从 2 开始（0, 1被哨兵占用）。
   - 堆下标从 1 开始（方便计算 `2x`, `2x+1`）。
   - 字符串哈希下标从 1 开始（前缀积计算方便）。
2. **边界条件**：
   - 栈/队列判空。
   - KMP 的 `j=0` 初始状态。
   - 并查集的初始化 `p[i]=i`。

### 代码风格
- **IO优化**：`ios::sync_with_stdio(false); cin.tie(0);`
- **数组大小**：`const int N = 100010;` 统一管理。
- **模板化**：熟练掌握 `init()`, `add()`, `query()` 等标准接口。
