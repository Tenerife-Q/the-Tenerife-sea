## 🎯 代码风格反思与优化

### 当前问题
- ❌ 过度美化的输入输出（菜单、提示语）导致代码臃肿
- ❌ 交互式设计增加复杂度，不符合算法竞赛场景
- ❌ 核心逻辑被包裹在大量UI代码中，可读性下降
- ❌ 对于参加算法比赛的目标来说，**得不偿失**

### 优化方案：双层结构

#### 层1：头部注释（学习用）
```cpp
/*
 * ============================================
 *   算法名称 - 核心原理
 *   题目来源
 * ============================================
 * 
 * 【问题】简述题意
 * 【思路】核心算法
 * 【图解】ASCII艺术/矩阵示意
 * 【复杂度】时间 + 空间
 * 【模板】核心代码框架
 */
```

#### 层2：精简代码（竞赛用）
```cpp
#include <iostream>
#include <algorithm>
using namespace std;

// 核心函数 + 行内注释
int solve() {
    // 关键步骤1
    // 关键步骤2
    return result;
}

int main() {
    // 标准输入输出，无菜单
    int n; cin >> n;
    // ...
    cout << ans << endl;
    return 0;
}
```

### 取舍原则
✅ **保留**：
- 详细的算法原理注释（头部）
- ASCII图解、数学推导（头部）
- 核心函数的行内注释（简短）

❌ **删除**：
- 交互式菜单（选择1/2/3）
- 详细演示模式（step-by-step输出）
- 中文提示语（"请输入..."）
- 多余的空行、分隔线

### 推荐模板风格
```cpp
/*
 * [算法名] - [核心技巧]
 * [题目] - [在线链接]
 * 
 * 思路：...
 * 复杂度：O(...)
 * 
 * 模板代码：
 * [粘贴核心3-5行]
 */

#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 10;
int n, a[N];

int main() {
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];
    
    // 核心算法
    sort(a, a + n);
    
    cout << a[0] << endl;
    return 0;
}
```

## 📚 第二章预告：数据结构

### 章节大纲（根据AcWing课程体系）

#### 1. 线性结构
- **单链表**：数组模拟，常用操作
- **双链表**：插入、删除优化
- **栈**：单调栈应用（表达式求值）
- **队列**：普通队列、循环队列
- **单调队列**：滑动窗口最值

#### 2. KMP 字符串匹配
- **next数组**：最长公共前后缀
- **模式匹配**：O(n+m)线性时间

#### 3. Trie 字典树
- **字符串集合**：快速查询前缀
- **最大异或对**：01-Trie应用

#### 4. 并查集 (Union-Find)
- **路径压缩**：find()优化
- **按秩合并**：union()优化
- **连通性问题**：图的动态维护

#### 5. 堆 (Heap)
- **小根堆/大根堆**：优先队列
- **堆排序**：O(n log n)
- **中位数维护**：对顶堆

#### 6. 哈希表 (Hash)
- **哈希函数**：字符串哈希
- **冲突处理**：拉链法、开放寻址
- **字符串匹配**：Rabin-Karp
