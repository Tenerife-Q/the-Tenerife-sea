这是一道经典的概率与动态规划（Dynamic Programming）结合的算法题。我会先为你**分析问题**，然后详细讲解背后的**数学原理**，最后给出解题思路。

---

### 第一部分：问题分析 (Problem Analysis)

**1. 题目目标**
投掷 $n$ 个标准的 6 面骰子，求所有骰子点数之和正好落在 $[a, b]$ 区间内的概率。
最后的结果需要保留 6 位小数（采用“四舍六入五成双/Banker's rounding”规则，这也是大多数编程语言中默认的浮点数格式化规则）。

**2. 数据规模分析**
*   $1 \le n \le 100$：最多投掷 100 个骰子。
*   $1 \le a \le b \le 6n$：点数之和最小是 $n$（全投 1），最大是 $6n$（全投 6）。

**3. 核心难点：为什么不能用“符合条件的方案数 / 总方案数”？**
在简单的概率题中，我们通常用：`概率 = 目标方案数 / 总方案数`。
但在这道题中，总方案数是 $6^n$。当 $n = 100$ 时，总方案数是 $6^{100}$，这是一个极大无比的数字，远远超出了 C++、Java 等语言中 64 位整型（`long long`）的存储范围。如果强行计算方案数，会导致整型溢出。

**4. 破局思路**
既然不能算“方案数”，我们可以直接在计算过程中推导**“概率”**。我们使用**动态规划 (DP)**，数组里面直接存储浮点数（`double`）概率值，这样就能完美避开大数溢出问题。

### 第二部分：数学原理讲解 (Mathematical Principles)

这道题背后依赖的核心数学原理是**古典概型**、**独立事件**以及**全概率公式**。

#### 1. 古典概型与独立事件 (Classical Probability & Independent Events)
对于单次投掷骰子，掷出 $1, 2, 3, 4, 5, 6$ 的概率是均等的，每一个点数的概率都是 $\frac{1}{6}$。
投掷第 $i$ 个骰子的结果，与前面 $i-1$ 个骰子的结果是**相互独立**的。这意味着它们的概率可以直接相乘。

#### 2. 全概率公式 (Law of Total Probability)
这是本题动态规划状态转移的核心数学依据。
假设我们想知道：“投掷 $i$ 个骰子，点数总和刚好为 $j$ 的概率是多少？” 记为 $P(i, j)$。

我们把第 $i$ 个骰子的投掷结果作为分类标准，它可能是 $1, 2, 3, 4, 5, 6$ 中的任意一个，并且每种结果的概率都是 $\frac{1}{6}$。
*   如果第 $i$ 个骰子投了 $1$，那么前 $i-1$ 个骰子的总和必须是 $j-1$。这种情况发生的概率是：$P(i-1, j-1) \times \frac{1}{6}$
*   如果第 $i$ 个骰子投了 $2$，那么前 $i-1$ 个骰子的总和必须是 $j-2$。这种情况发生的概率是：$P(i-1, j-2) \times \frac{1}{6}$
*   ... 以此类推。

根据全概率公式，我们将所有互斥的情况加起来，就得到了递推公式：
$$P(i, j) = \sum_{k=1}^{6} P(i-1, j-k) \times \frac{1}{6}$$
*(前提是 $j-k \ge 0$)*

#### 3. 数学期望与概率分布的演变
如果你画出 $P(n, j)$ 的图像：
*   当 $n=1$ 时，概率分布是一个均匀的矩形（1到6的概率都是 $1/6$）。
*   当 $n=2$ 时，概率分布变成了一个三角形（和为7的概率最大，两端概率最小）。
*   当 $n$ 逐渐增大（比如 $n \ge 10$），根据**中心极限定理 (Central Limit Theorem)**，骰子和的概率分布会越来越接近于**正态分布 (Normal Distribution)**的钟形曲线。这也是为什么题目要求精确到 6 位小数，因为极端值（如全是 1）的概率在 $n$ 变大后会小到几乎为 0。

### 第三部分：算法设计 (Algorithm Design)

结合上面的数学原理，我们可以设计出如下的 DP 算法：

**1. 状态定义：**
设 `dp[i][j]` 表示投掷了 `i` 个骰子后，点数之和恰好为 `j` 的概率。

**2. 初始状态 (Base Case)：**
`dp[0][0] = 1.0`
（投掷 0 个骰子，和为 0 的概率是 100%。其他所有 `dp[0][j] = 0`）。

**3. 状态转移方程：**
对于每一个骰子个数 `i` (从 1 到 $n$)：
对于每一个可能的和 `j` (从 `i` 到 $6i$)：
对于第 `i` 次可能掷出的点数 `k` (从 1 到 6)：
$$dp[i][j] = dp[i][j] + \frac{dp[i-1][j-k]}{6.0}$$
*(注意边界检查：只有当 $j - k \ge 0$ 时才进行累加)*

**4. 计算最终结果：**
题目要求求和在 $[a, b]$ 区间内的概率。
所以最终答案就是把所有符合条件的概率加起来：
$$Answer = \sum_{x=a}^{b} dp[n][x]$$

**5. 空间优化（可选）：**
观察状态转移方程可以看出，第 `i` 层的状态只依赖于第 `i-1` 层的状态。因此，我们可以用**滚动数组**（两个一维数组）来代替二维数组，从而将空间复杂度从 $O(n \times 6n)$ 降低到 $O(6n)$。不过本题 $n \le 100$，二维数组 $100 \times 600$ 完全不会超出内存限制，可以直接写二维数组。

### 总结
这道题是**全概率公式**在计算机算法中的经典应用。通过将“求组合数”转化为“求概率的动态规划”，我们巧妙地避开了组合爆炸带来的大数溢出问题。最终的时间复杂度为 $O(n \times 6n \times 6) \approx O(n^2)$，对于 $n=100$ 的数据规模，计算量仅为几万次，在 1.00s 的时间限制内可以瞬间完成。

### 第一部分：数学推导与实例证明

#### 1. 数学模型定义
设 $S_i$ 表示投掷 $i$ 个骰子的点数总和。
我们要计算的是事件 $\{S_i = j\}$ 发生的概率，记为 $P(S_i = j)$。
为了方便在程序中表示，我们记 $dp[i][j] = P(S_i = j)$。

#### 2. 全概率公式推导递推关系 (状态转移)
假设我们已经知道了投掷 $i-1$ 个骰子时，产生各种和的概率。现在投掷第 $i$ 个骰子。
第 $i$ 个骰子的结果记为 $X_i$，它的取值只可能是 $\{1, 2, 3, 4, 5, 6\}$，且每种取值的概率都是 $\frac{1}{6}$，即 $P(X_i = k) = \frac{1}{6}$。

由于**每次投掷是相互独立**的，要求前 $i$ 个骰子的和为 $j$ ($S_i = j$)，我们可以根据第 $i$ 个骰子的点数进行分类：
*   如果 $X_i = 1$，那么前 $i-1$ 个骰子的和必须是 $j-1$。
*   如果 $X_i = 2$，那么前 $i-1$ 个骰子的和必须是 $j-2$。
*   ...
*   如果 $X_i = k$，那么前 $i-1$ 个骰子的和必须是 $j-k$。

根据**全概率公式**（Law of Total Probability）：
$$ P(S_i = j) = \sum_{k=1}^{6} P(X_i = k) \cdot P(S_{i-1} = j - k) $$
因为 $P(X_i = k) = \frac{1}{6}$，代入得到：
$$ dp[i][j] = \sum_{k=1}^{6} \frac{1}{6} \cdot dp[i-1][j-k] $$
*(前提是 $j-k \ge 0$ 且 $j-k \le 6(i-1)$)*

#### 3. 实例详细计算证明（对应题目 Example）
题目样例：**Input:** `2 9 10` (即 $n=2$ 个骰子，求和在 $a=9$ 到 $b=10$ 之间的概率)

**Step 1：初始化 ($i=0$)**
没有骰子时和为0的概率为100%：$dp[0][0] = 1$。

**Step 2：投第1个骰子 ($i=1$)**
根据公式计算：
$dp[1][1] = \frac{1}{6} \cdot dp[0][0] = \frac{1}{6}$
同理，$dp[1][2] = dp[1][3] = dp[1][4] = dp[1][5] = dp[1][6] = \frac{1}{6}$。

**Step 3：投第2个骰子 ($i=2$)**
题目要求求出和为 9 和 10 的概率。我们只需计算 $dp[2][9]$ 和 $dp[2][10]$：

*   计算 $dp[2][9]$：
    $$ dp[2][9] = \frac{1}{6} \times (dp[1][8] + dp[1][7] + dp[1][6] + dp[1][5] + dp[1][4] + dp[1][3]) $$
    因为 1 个骰子不可能投出 7 和 8，所以 $dp[1][8] = dp[1][7] = 0$。
    $$ dp[2][9] = \frac{1}{6} \times (0 + 0 + \frac{1}{6} + \frac{1}{6} + \frac{1}{6} + \frac{1}{6}) = \frac{1}{6} \times \frac{4}{6} = \frac{4}{36} $$
    *(直观理解：组合有 (3,6), (4,5), (5,4), (6,3)，共4种)*

*   计算 $dp[2][10]$：
    $$ dp[2][10] = \frac{1}{6} \times (dp[1][9] + dp[1][8] + dp[1][7] + dp[1][6] + dp[1][5] + dp[1][4]) $$
    只有 $dp[1][6]$、$dp[1][5]$、$dp[1][4]$ 是有概率的。
    $$ dp[2][10] = \frac{1}{6} \times (0 + 0 + 0 + \frac{1}{6} + \frac{1}{6} + \frac{1}{6}) = \frac{1}{6} \times \frac{3}{6} = \frac{3}{36} $$
    *(直观理解：组合有 (4,6), (5,5), (6,4)，共3种)*

**Step 4：求总概率**
区间 $[9, 10]$ 的总概率为：
$$ P(9 \le S_2 \le 10) = dp[2][9] + dp[2][10] = \frac{4}{36} + \frac{3}{36} = \frac{7}{36} $$
我们把 $\frac{7}{36}$ 转化为小数：$7 \div 36 = 0.19444444...$
保留 6 位小数，结果刚好为 **`0.194444`**。
这就完美证明了题目 Example 的正确性以及我们动态规划状态转移方程的正确性。

### 第二部分：C++ 代码实现

代码逻辑与上面的推导完全一致。我们直接使用 `double` 类型的二维数组来记录概率。

```cpp
#include <iostream>
#include <vector>
#include <iomanip> // 用于格式化浮点数输出

using namespace std;

int main() {
    // 优化输入输出速度
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n, a, b;
    if (!(cin >> n >> a >> b)) return 0;

    // dp[i][j] 表示投掷 i 个骰子后，点数之和为 j 的概率
    // i 的最大值是 n，j 的最大值是 6 * n
    vector<vector<double>> dp(n + 1, vector<double>(6 * n + 1, 0.0));

    // 初始状态 (Base Case)
    // 投掷 0 个骰子，和为 0 的概率是 100% (即 1.0)
    dp[0][0] = 1.0;

    // 动态规划：外层循环遍历骰子个数 1 到 n
    for (int i = 1; i <= n; i++) {
        // 中层循环遍历当前可能产生的所有点数和
        // i 个骰子的和最小是 i，最大是 6 * i
        for (int j = i; j <= 6 * i; j++) {
            // 内层循环遍历当前第 i 个骰子可能投出的点数 (1 到 6)
            for (int k = 1; k <= 6; k++) {
                // 如果前 i-1 个骰子的点数和合法 (大于等于 0)
                if (j - k >= 0) {
                    // 全概率公式累加
                    dp[i][j] += dp[i - 1][j - k] / 6.0;
                }
            }
        }
    }

    // 计算最终结果：把 [a, b] 范围内的概率加起来
    double result = 0.0;
    for (int j = a; j <= b; j++) {
        result += dp[n][j];
    }

    // 输出结果，保留 6 位小数
    cout << fixed << setprecision(6) << result << "\n";

    return 0;
}
```

### 第三部分：复杂度分析
*   **时间复杂度**：三重循环。外层 $O(n)$，中层 $O(6n)$，内层 $O(6)$（常数）。总时间复杂度为 $O(n \times 6n \times 6) = O(n^2)$。当 $n=100$ 时，循环次数仅仅约为 $100 \times 600 \times 6 = 360,000$ 次，对于现代计算机（每秒运行 $10^8$ 次运算）来说，执行时间远小于 1 毫秒，轻松通过 1.00s 的限制。
*   **空间复杂度**：我们申请了一个大小为 $(n+1) \times (6n+1)$ 的 `double` 类型数组。当 $n=100$ 时，数组大小约为 $101 \times 601$ 个浮点数，占用内存不到 `500 KB`，远远低于题目给出的 `512 MB` 的内存限制。因此使用二维数组是最清晰且完全合理的选择。
