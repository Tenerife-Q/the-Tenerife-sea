


既然代码的实现本身就是对公式的直接套用，那么真正吃透这道题的关键，就在于彻底理解**卡特兰数（Catalan Number）底层推导的数学逻辑**。

为了让你有醍醐灌顶的感觉，我们采用**“数形结合”（几何变换 + 代数推导）**的方法，从零开始详细剖析这个公式是怎么来的。

---

### 一、 数形结合：将“括号”转化为“走格子”

要搞懂括号匹配，我们首先建立一个直角坐标系。
假设题目输入的总长度为 $2n$。要成为合法的括号序列，必须有 $n$ 个左括号 `(` 和 $n$ 个右括号 `)`。

**1. 建立坐标系规则**
- 初始点在原点 $(0, 0)$。
- 遇到一个**左括号 `(`**，向**右**走一步，即 $X$ 坐标 $+1$。
- 遇到一个**右括号 `)`**，向**上**走一步，即 $Y$ 坐标 $+1$。

**2. 终点在哪？**
无论括号顺序如何，因为总共有 $n$ 个左括号和 $n$ 个右括号，走完 $2n$ 步后，**终点必定是 $(n, n)$**。

**3. “总方案数”是多少？**
从 $(0,0)$ 走到 $(n,n)$，一共需要走 $2n$ 步，其中有 $n$ 步必须向右走。
所以，没有任何限制情况下的**总走法数**，就是在 $2n$ 步里挑出 $n$ 步向右走（剩下的自动向上走），即组合数：
$$ \text{总方案数} = \binom{2n}{n} $$

---

### 二、 几何边界：寻找“不合法序列”的特征

合法序列的条件是：**对于任意前缀，右括号的数量绝不能超过左括号的数量。**
把它翻译到我们的坐标系里：
- 右边走的步数（$X$）必须始终大于等于向上走的步数（$Y$）。
- 即，路径上的任意一点 $(X,Y)$ 都必须满足 $X \ge Y$。
- 这意味着，**合法的路径永远不能越过对角线 $y = x$**。

那么，什么是**不合法**的路径呢？
一旦路径越过了 $y = x$，它必然会触碰或穿过上面挨着的一条平行线：**$y = x + 1$**。
所以，“包含不合法括号序列的路径”的几何定义就是：**从 $(0,0)$ 走到 $(n,n)$，并且中途触碰过直线 $y = x + 1$ 的路径。**

---

### 三、 镜像反射原理（André's Reflection Principle）

这是整个推导中最妙的一步：怎么计算这些触碰过 $y = x + 1$ 的不合法路径有多少条？我们使用**“翻转/镜像法”**。

1. **设置一面镜子**：把直线 $y = x + 1$ 想象成一面镜子。
2. **发生反射**：对于任意一条不合法的路径，它从 $(0,0)$ 出发，一定会**第一次**撞上这面镜子（直线 $y = x + 1$）。
3. **折叠路径**：我们把撞上镜子这一刻**之后**的所有路径，沿着这面镜子作对称翻转。
4. **终点变了**：
   原路径的终点是 $(n, n)$。
   根据解析几何，点 $(X,Y)$ 沿着直线 $y = x + c$ 对称的镜像点是 $(Y-c, X+c)$。
   那么终点 $(n, n)$ 沿着直线 $y = x + 1$ 翻转后，新终点变成了 $(n-1, n+1)$。

**结论（一一对应关系）：**
每一条原本通向 $(n,n)$ 的不合法路径，经过翻转后，都**变成了一条从 $(0,0)$ 通向 $(n-1, n+1)$ 的路径**。
反之，任何一条从 $(0,0)$ 走到 $(n-1, n+1)$ 的路径，因为终点在 $y=x+1$ 的上方，必定穿过这面镜子，把它翻转回来，就能得到一条不合法路径。

所以：
**不合法路径的数量** = **从 $(0,0)$ 走到 $(n-1, n+1)$ 的所有可能路径数量**。

从 $(0,0)$ 走到 $(n-1, n+1)$，总共走的步数还是 $(n-1) + (n+1) = 2n$ 步。其中向右走的步数是 $n-1$ 步。
所以，不合法路径数为：
$$ \text{不合法方案数} = \binom{2n}{n-1} $$

---

### 四、 代数推导：得到优雅的卡特兰数公式

现在，我们用“总方案数”减去“不合法方案数”，就得到了“合法方案数”（即卡特兰数 $C_n$）：

$$ C_n = \binom{2n}{n} - \binom{2n}{n-1} $$

这一步其实就是程序的原始逻辑。但为了让代码跑得更快（也为了防溢出），我们进行代数化简，将它变成单项乘积的形式：

根据组合数定义 $\binom{N}{K} = \frac{N!}{K!(N-K)!}$，展开上述公式：

$$ C_n = \frac{(2n)!}{n! \cdot n!} - \frac{(2n)!}{(n-1)! \cdot (n+1)!} $$

提取公因式 $\frac{(2n)!}{n! \cdot n!}$。注意后半部分的 $(n+1)!$ 可以拆成 $(n+1) \cdot n!$，而 $n!$ 可以拆成 $n \cdot (n-1)!$，所以：

$$ C_n = \frac{(2n)!}{n! \cdot n!} \left( 1 - \frac{n}{n+1} \right) $$

括号里通分计算：$1 - \frac{n}{n+1} = \frac{n+1}{n+1} - \frac{n}{n+1} = \frac{1}{n+1}$

最后乘回去，得到最终极其简洁的通项公式：

$$ C_n = \frac{1}{n+1} \binom{2n}{n} = \frac{(2n)!}{(n+1)! \cdot n!} $$

这就是**为什么代码里只需要计算 $(2n)!$ 并除以 $n!$ 和 $(n+1)!$ 的底层原理**！理解了这里，再看代码就会感到十分自然和透彻。

### 完整输出可复制的代码文件

```cpp
// 标题：Bracket Sequences I - 基于卡特兰数公式 (2n)! / (n! * (n+1)!)
#include <iostream>

using namespace std;

// 题目要求的取模素数
const int MOD = 1000000007;

// 快速幂算法：计算 (base^exp) % MOD
// 作用：用于配合费马小定理求逆元
long long modPow(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

// 费马小定理求逆元：求 a 在模 p 下的逆元，等于 a^(p-2) % p
// 作用：模运算中没有除法，除以一个数等价于乘以它的逆元
long long modInverse(long long n) {
    return modPow(n, MOD - 2);
}

int main() {
    // 优化 C++ 的标准输入输出速度
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;

    // 奇数个字符无法做到左右括号成对匹配
    if (n % 2 != 0) {
        cout << 0 << "\n";
        return 0;
    }

    // 题目长度 n 即为推导公式里的 2k
    // 这里的 k 代表左/右括号各自的数量
    int k = n / 2;

    long long fact_2k = 1;  // 用于存储分子 (2k)!
    long long fact_k = 1;   // 用于存储分母的一部分 k!
    long long fact_k1 = 1;  // 用于存储分母的另一部分 (k+1)!

    // 遍历一次，计算阶乘并随时取模以防溢出
    for (int i = 1; i <= n; i++) {
        fact_2k = (fact_2k * i) % MOD;
        
        // 当遍历到 k 时，把当前结果记下来，作为 k!
        if (i == k) {
            fact_k = fact_2k;
        }
        // 当遍历到 k+1 时，把当前结果记下来，作为 (k+1)!
        if (i == k + 1) {
            fact_k1 = fact_2k;
        }
    }

    // 卡特兰数代数化简后的最终公式分母部分：k! * (k+1)!
    long long denominator = (fact_k * fact_k1) % MOD;

    // 最终结果：分子 * 分母的逆元 % MOD
    long long ans = (fact_2k * modInverse(denominator)) % MOD;

    // 输出最终合法序列方案数
    cout << ans << "\n";

    return 0;
}
```




这份官方题解的核心逻辑与我们前面讲的**卡特兰数（Catalan Number）**完全一致！

只不过，官方题解在推导“不合法方案数”时，没有使用二维坐标系的“折线对称法”，而是使用了**“括号反转法”（序列翻转）**。此外，官方题解在代数化简时，直接保留了总长度 $n$，得出了一个等价但形式略有不同的公式：$\frac{2}{n+2} \binom{n}{n/2}$。

为了让你彻底吃透这份标答，我们按步骤对其数学逻辑和代码实现进行详细拆解。

---

### 一、 标答的数学原理解析

#### 1. 总方案数
假设序列总长度为 $n$（$n$ 必须是偶数）。我们要填入 $\frac{n}{2}$ 个左括号和 $\frac{n}{2}$ 个右括号。
没有任何限制的情况下，从 $n$ 个位置中挑出 $\frac{n}{2}$ 个位置放左括号，总方案数是：
$$ \text{总方案数} = \binom{n}{\frac{n}{2}} $$

#### 2. 寻找“不合法方案数”（括号反转法）
标答中提到：**一个序列不合法，意味着它的某个前缀里，右括号的数量超过了左括号。**
我们找到**最短的那个不合法前缀**（Minimal invalid prefix）。在这个最短前缀中，右括号一定刚好比左括号多 1 个。

**标答的巧妙操作：**
我们将这个最短前缀里的所有括号**全部反转**（即左括号变成右括号，右括号变成左括号）。
- 反转前：前缀中有 $x$ 个左括号，$x+1$ 个右括号。
- 反转后：前缀变成了 $x+1$ 个左括号，$x$ 个右括号。

**反转后，整个序列的括号数量发生了什么变化？**
左括号多出了 1 个，右括号减少了 1 个。
所以，原本由 $\frac{n}{2}$ 个左括号和 $\frac{n}{2}$ 个右括号组成的不合法序列，经过这样一一对应的反转后，全都变成了一个由 **$\frac{n}{2} + 1$ 个左括号** 和 **$\frac{n}{2} - 1$ 个右括号** 组成的序列！

> **⚠️ 注意标答文本里的一个笔误（Typo）：**
> 标答原文写道：“...we get another bracket sequence that consists of **n+1** left brackets and **n-1** right brackets.”
> 这是一个明显的笔误，总长度才 $n$，不可能有 $n+1$ 个左括号。它想表达的其实是 **$\frac{n}{2}+1$** 个左括号和 **$\frac{n}{2}-1$** 个右括号。你看它下一行的公式就写对了。

所以，不合法方案的数量等于：
$$ \text{不合法方案数} = \binom{n}{\frac{n}{2} + 1} $$

#### 3. 代数化简（从减法变成乘法）
合法方案数 = 总方案数 - 不合法方案数。标答给出的公式是：
$$ \text{答案} = \binom{n}{\frac{n}{2}} - \binom{n}{\frac{n}{2} + 1} $$

为了让程序算得更快，标答对这个式子进行了极其漂亮的化简。我们令 $k = \frac{n}{2}$：
$$ \binom{2k}{k} - \binom{2k}{k+1} $$
$$ = \frac{(2k)!}{k! \cdot k!} - \frac{(2k)!}{(k+1)! \cdot (k-1)!} $$
提取公因式 $\frac{(2k)!}{k! \cdot k!}$，也就是提取出 $\binom{2k}{k}$：
$$ = \binom{2k}{k} \left( 1 - \frac{k}{k+1} \right) $$
$$ = \binom{2k}{k} \left( \frac{1}{k+1} \right) $$

把 $k = \frac{n}{2}$ 代回到 $\frac{1}{k+1}$ 中：
$$ \frac{1}{\frac{n}{2} + 1} = \frac{1}{\frac{n+2}{2}} = \frac{2}{n+2} $$

所以，最终化简出来的公式就是：
$$ \text{答案} = \frac{2}{n+2} \binom{n}{\frac{n}{2}} $$
这就是代码里核心计算逻辑的由来！

---

### 二、 标答代码拆解

标答的代码完全是围绕着公式 $\frac{2}{n+2} \binom{n}{n/2}$ 来写的。这种写法在竞技编程（CP）中非常标准。

#### 1. 预处理阶乘数组
```cpp
const int N = 1000000;
ll fac[N + 1];

fac[0] = 1;
for (int i = 1; i <= N; i++) {
    fac[i] = fac[i - 1] * i % M;
}
```
因为组合数公式 $\binom{n}{k} = \frac{n!}{k!(n-k)!}$ 强依赖阶乘，标答直接开了一个大小为 $10^6$ 的数组 `fac`，在 `main` 函数开头把 $1$ 到 $10^6$ 的所有阶乘都算好存起来。
**好处**：后续任何求阶乘的操作都是 $O(1)$ 复杂度。

#### 2. 快速幂与逆元
```cpp
int power(int a, int b) { ... } // 快速幂 a^b % M
int inv(int x) { return power(x, M - 2); } // 费马小定理求逆元 x^(M-2) % M
```
这和我们之前的逻辑一样。模运算下的除法必须转换为乘以逆元。

#### 3. 封装组合数函数 `ncr`
```cpp
int ncr(int a, int b) {
    return fac[a] * inv(fac[b] * fac[a - b] % M) % M;
}
```
`ncr(a, b)` 也就是求 $\binom{a}{b}$。
对应公式 $\frac{a!}{b!(a-b)!}$，代码里写作：`fac[a] * inv( fac[b] * fac[a-b] )`。

#### 4. 核心主逻辑
```cpp
if (n % 2 == 0) {
    ll answer = 2LL * ncr(n, n / 2) * inv(n + 2) % M;
    cout << answer << "\n";
}
```
这里完美复刻了推导出的公式：
$$ \text{答案} = 2 \times \binom{n}{n/2} \times (n+2)^{-1} \pmod M $$
`2LL` 是为了防止乘法过程中的整型溢出（强制转换为 `long long`）。
这行代码相当于：先算 $2 \times \binom{n}{n/2}$，然后再除以 $(n+2)$（即乘以 `n+2` 的逆元）。

### 总结

- **思维方式差异**：我们前文讲的“数形结合”是把括号转换成“二维坐标系走的网格路径”作镜像反转；而标答是直接在“字符串序列”上对前缀进行“字符反转”。两者在数学本质上是**完全等价**的（镜像反转本质就是互换 $X$ 和 $Y$ 前进方向，也就是互换左右括号）。
- **公式形态差异**：我们提取了 $k=n/2$，公式是 $\frac{1}{k+1} \binom{2k}{k}$；标答保留了 $n$，公式是 $\frac{2}{n+2} \binom{n}{n/2}$。代数上也是**完全相等**的。
- **代码风格差异**：标答采用了“预处理所有阶乘”的工程化写法，把求组合数 `ncr` 封装成了独立函数，代码看起来更加模块化，也更容易扩展应对“多组测试用例（Multiple Queries）”的情况。
