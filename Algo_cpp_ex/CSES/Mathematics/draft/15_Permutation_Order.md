# 排列顺序 (Permutation Order) 题解与学习笔记

## 📌 题目概述

**题目背景：**
令 $p(n,k)$ 表示包含数字 $1 \dots n$ 且按**字典序**排列的第 $k$ 个排列。
例如，当 $n=4$ 时，$p(4,1) = [1,2,3,4]$，$p(4,2) = [1,2,4,3]$。

**任务：**
处理以下两种类型的操作（数据范围：$n \le 20$）：
1. **已知序号求排列**：给定 $n$ 和 $k$，求出对应的排列 $p(n,k)$。
2. **已知排列求序号**：给定 $n$ 和排列 $p(n,k)$，求出对应的序号 $k$。

---

## 🧮 核心数学原理：康托展开与逆康托展开

本题本质上是经典的组合数学模型：**康托展开（Cantor Expansion）**。
核心思想是：**从左到右逐位确定数字，每一位的选择取决于它后面剩余位置的全排列总数（阶乘）。**

> 💡 **关键统一：** 为了数学计算的连贯性，我们在进行除法和取余运算时，必须将序号转化为 **0-indexed（从 0 开始计数的索引）**。即第 1 个排列的内部序号为 0。

### 1. 操作一：已知 $k$ 求排列（逆康托展开）
假设我们有一个装满未使用数字的**候选集合** $S = \{1, 2, \dots, n\}$。
对于排列的第 $i$ 位（$0 \le i < n$），它后面还剩下 $(n-1-i)$ 个空位，这意味着只要我们确定了当前这一位的数字，后面就会有 $(n-1-i)!$ 种排列方式（我们称之为“一个大类”）。

* **选数字**：我们需要跳过 $\lfloor k / (n-1-i)! \rfloor$ 个大类。因此，当前位应该填入候选集合 $S$ 中第 $idx = k / (n-1-i)!$ 小的数字（0-indexed）。
* **更新状态**：将选中的数字从 $S$ 中移除，更新剩下需要跳过的排列数 $k = k \pmod{(n-1-i)!}$。

### 2. 操作二：已知排列求 $k$（康托展开）
同样维护一个**候选集合** $S = \{1, 2, \dots, n\}$。设初始跳过的排列数 $k = 0$。
遍历给定排列的每一位数字 $x$：

* **查排位**：在当前的候选集合 $S$ 中，查找有几个数字比 $x$ 小，设为 $idx$。
* **累加跳过数量**：这说明在我们遇到 $x$ 之前，已经完整跳过了 $idx$ 个大类。因此累加 $k = k + idx \times (n-1-i)!$。
* **更新状态**：将 $x$ 从 $S$ 中移除。遍历结束后，$k + 1$ 即为最终的 1-indexed 序号。

## 📝 流程推导实例演练

以 $n=4$ 为例，我们有初始候选集合 $S = \{1, 2, 3, 4\}$。

**场景：已知排列 `[2, 4, 1, 3]`，求它是第几个排列？**
1. **第 1 位 `2`**：集合 $S=\{1,2,3,4\}$ 中，只有 `1` 比它小（$idx=1$）。跳过 $1 \times 3! = 6$ 个排列。移除 `2`，剩余 $S=\{1,3,4\}$。
2. **第 2 位 `4`**：集合 $S=\{1,3,4\}$ 中，`1, 3` 比它小（$idx=2$）。跳过 $2 \times 2! = 4$ 个排列。移除 `4`，剩余 $S=\{1,3\}$。
3. **第 3 位 `1`**：集合 $S=\{1,3\}$ 中，没有比它小的（$idx=0$）。跳过 $0 \times 1! = 0$ 个排列。移除 `1`，剩余 $S=\{3\}$。
4. **第 4 位 `3`**：跳过 $0 \times 0! = 0$ 个排列。

**总共跳过**：$6 + 4 + 0 + 0 = 10$ 个排列。
对应的 0-indexed 序号为 $10$，转换回 1-indexed 即为 **第 11 个排列**。

*(逆向推导“已知第 11 个求排列”时，只需不断用 $10$ 去整除和取余对应的阶乘即可，过程完全对称。)*

## 💻 究极融合版 C++ 题解

本代码在纯粹的数学逻辑基础上，完美融合了 C++ STL（标准模板库）的高级技巧：
1. **`std::iota`**：一键完成递增数组初始化，替代繁琐的 `for` 循环。
2. **`std::find` 与指针减法**：利用迭代器减法 `it - numbers.begin()` 一步到位算出“有几个更小的可用数字被跳过了”，极其优雅。
3. **直观的阶乘查表**：保留 `fact[n-1-i]` 查表法，让代码与数学公式完全对应，极大增强了可读性。

```cpp
#include <iostream>
#include <vector>
#include <numeric>   // 提供 std::iota
#include <algorithm> // 提供 std::find

using namespace std;
using ll = long long;

const int N = 20;
ll fact[N + 1];

// 预处理阶乘数组 (0! 到 20!)
// 20! 约为 2.43 * 10^18，在 64 位有符号整数 (long long) 的安全范围内
void precompute() {
    fact[0] = 1;
    for (int i = 1; i <= N; i++) {
        fact[i] = fact[i - 1] * i;
    }
}

void solve() {
    int op, n;
    cin >> op >> n;

    // 优雅地初始化候选数字集合 {1, 2, ..., n}
    vector<int> numbers(n);
    iota(numbers.begin(), numbers.end(), 1); 

    if (op == 1) {
        // --- 操作 1: 已知序号 k 求排列 (逆康托展开) ---
        ll k;
        cin >> k;
        k--; // 转换为 0-indexed 以便进行除法和取余运算

        for (int i = 0; i < n; i++) {
            ll f = fact[n - 1 - i]; // 获取当前位对应的“大类”排列数
            
            int idx = k / f; // 计算应当选取候选集中排第几的数字(即下标)
            cout << numbers[idx] << (i == n - 1 ? "" : " ");
            
            numbers.erase(numbers.begin() + idx); // 从候选集中移除已用数字
            k %= f; // 更新剩余需要跳过的排列数
        }
        cout << "\n";
        
    } else if (op == 2) {
        // --- 操作 2: 已知排列求序号 k (康托展开) ---
        ll k = 1; // 直接初始化为 1-indexed，最终无需再 +1

        for (int i = 0; i < n; i++) {
            int x;
            cin >> x;
            
            ll f = fact[n - 1 - i]; // 获取当前位对应的“大类”排列数
            
            // 使用 find 寻找当前数字 x 在候选集中的位置(迭代器)
            auto it = find(numbers.begin(), numbers.end(), x);
            
            // 迭代器相减得到下标 idx，它完美等价于：“集合中有几个可用数字比 x 小”
            int idx = it - numbers.begin(); 
            
            k += idx * f; // 累加跳过的大类排列数
            numbers.erase(it); // 从候选集中移除处理过的数字
        }
        cout << k << "\n";
    }
}

int main() {
    // 优化 C++ 的输入输出流速度，防止在多组数据时超时
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    precompute();

    int t;
    if (cin >> t) {
        while (t--) {
            solve();
        }
    }
    return 0;
}
```

## ⚙️ 复杂度分析

- **时间复杂度**：
  - 单次查询：遍历 $n$ 位数字，每次内部调用 `std::erase` 和 `std::find` 均耗时 $O(n)$，故单次操作时间复杂度为 $\mathcal{O}(n^2)$。
  - 总体：对于 $T$ 组测试用例，总时间复杂度为 $\mathcal{O}(T \cdot n^2)$。由于 $n \le 20, T \le 1000$，最大运算量约为 $4 \times 10^5$ 次操作，在 1.00s 时限内瞬间执行完毕。
- **空间复杂度**：需要存储阶乘的数组和临时的 `vector`，空间复杂度为 $\mathcal{O}(n)$，极其轻量，远低于 512 MB 的内存限制。
