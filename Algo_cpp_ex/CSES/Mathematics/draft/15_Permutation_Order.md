# Permutation Order (排列顺序) 题解与学习笔记

## 📌 题目概述

**题目背景：**
令 $p(n,k)$ 表示包含数字 $1 \dots n$ 且按 **字典序** 排列的第 $k$ 个排列。
例如，当 $n=4$ 时：
- $p(4,1) =[1,2,3,4]$
- $p(4,2) =[1,2,4,3]$

**任务：**
处理以下两种类型的操作（数据范围：$n \le 20$）：
1. **已知序号求排列**：给定 $n$ 和 $k$，求出对应的排列 $p(n,k)$。
2. **已知排列求序号**：给定 $n$ 和排列 $p(n,k)$，求出对应的序号 $k$。

---

## 🧮 核心数学原理：康托展开与逆康托展开

本题本质上是经典的组合数学模型：**康托展开（Cantor Expansion）**。
它的核心思想是：**从左到右逐位确定数字，每一位的选择取决于它后面剩余位置的全排列总数（即阶乘）。**

> 💡 **关键点：0-indexed（从 0 开始计数）**
> 为了数学取余和整除计算的连贯性，我们在推导时必须将“第几个”转化为“跳过了几个”。即第 1 个排列的内部序号为 0，第 11 个排列的内部序号为 10。

---

## 📝 详细流程推导（以 $n=4$ 为例）

我们维护一个 **“尚未被使用的候选数字集合”** $S$。初始时 $S = \{1, 2, 3, 4\}$。

### 场景一：已知排列 `[2, 4, 1, 3]`，求它是第几个排列？（康托展开）
**规则**：从左到右遍历排列，看当前数字在 $S$ 中排第几（即有几个比它小的可用数字），这意味着我们 **“完整跳过了几个大类”**。

| 步骤 | 当前位数字 | 剩余候选集合 $S$ | 有几个比它小的数字 ($idx$) | 后面剩余空位 | 跳过的排列数 ($idx \times 阶乘$) | 累积跳过总数 $k$ | 更新后的 $S$ (移除当前数字) |
| :---: | :---: | :--- | :---: | :---: | :--- | :--- | :--- |
| **1** | `2` | `{1, 2, 3, 4}` | `1` (只有1) | $3$ | $1 \times 3! = 6$ | $0 + 6 = 6$ | `{1, 3, 4}` |
| **2** | `4` | `{1, 3, 4}` | `2` (有1,3) | $2$ | $2 \times 2! = 4$ | $6 + 4 = 10$ | `{1, 3}` |
| **3** | `1` | `{1, 3}` | `0` (没有) | $1$ | $0 \times 1! = 0$ | $10 + 0 = 10$ | `{3}` |
| **4** | `3` | `{3}` | `0` (没有) | $0$ | $0 \times 0! = 0$ | $10 + 0 = 10$ | `{}` |

**结论**：一共跳过了 $10$ 个排列（0-indexed 为 10）。转换回日常的 1-indexed，它是 **第 11 个排列**。

### 场景二：已知求 $n=4$ 的第 11 个排列是什么？（逆康托展开）
**规则**：将序号转为跳过数（$k = 11 - 1 = 10$）。用 $k$ 去除以后面空位产生的排列数（阶乘），**商**代表我们要从 $S$ 中选第几个数字，**余数**代表选完后还剩下多少个排列需要跳过。

| 步骤 | 剩余空位 | 当前阶乘 $f$ | 剩余要跳过的 $k$ | 商 $idx = k / f$ (选 $S$ 中第 $idx$ 个) | 当前候选集合 $S$ | 选中的数字 | 余数 $k = k \pmod f$ | 更新后的 $S$ (移除选中数字) |
| :---: | :---: | :---: | :---: | :---: | :--- | :---: | :---: | :--- |
| **1** | $3$ | $3! = 6$ | $10$ | $10 / 6 = 1$ | `{1, 2, 3, 4}` | **`2`** | $10 \pmod 6 = 4$ | `{1, 3, 4}` |
| **2** | $2$ | $2! = 2$ | $4$ | $4 / 2 = 2$ | `{1, 3, 4}` | **`4`** | $4 \pmod 2 = 0$ | `{1, 3}` |
| **3** | $1$ | $1! = 1$ | $0$ | $0 / 1 = 0$ | `{1, 3}` | **`1`** | $0 \pmod 1 = 0$ | `{3}` |
| **4** | $0$ | $0! = 1$ | $0$ | $0 / 1 = 0$ | `{3}` | **`3`** | $0 \pmod 1 = 0$ | `{}` |

**结论**：拼接选中的数字，排列为 **`[2, 4, 1, 3]`**。逻辑与场景一完美对称闭环。

## 💡 代码演进与原理解析 (融合官方标答)

在实现代码时，我们面临如何维护“集合 $S$”和“阶乘计算”的问题。结合常规思路和官方标答，我们做了以下 **取其精华** 的融合：

1. **关于集合 $S$ 的初始化：**
   - **常规写法**：用 `for` 循环 `numbers[i] = i + 1`。
   - **官方技巧**：使用 `<numeric>` 库的 `iota(numbers.begin(), numbers.end(), 1)`。一句话将数组填满递增数字，极其优雅，**我们在最终代码中采用此法**。

2. **关于“查找数字在 $S$ 中的位置并统计比它小的个数”：**
   - **常规写法**：写一个内部 `for` 循环，遍历寻找匹配的元素下标。
   - **官方技巧**：使用 `<algorithm>` 库的 `find` 返回迭代器（指针），利用指针减法 `it - numbers.begin()` 直接获取下标。因为 $S$ 始终是有序的，**这个下标恰好等于“有几个可用的数字比它小”**。这堪称神来之笔，**我们在最终代码中强烈推荐此法**。

3. **关于“获取剩余空位的阶乘”：**
   - **官方技巧**：不预处理数组，而是初始令 `f = n!`，每次循环执行 `f /= (n - i)` 动态降级。
   - **常规写法（优化采用）**：预处理 `fact` 阶乘数组，每次直接用 `fact[n - 1 - i]`。
   - **抉择**：官方的除法虽然巧妙，但掩盖了 $(n-1-i)!$ 的数学本质。为了**代码可读性与数学公式的直观对应**，我们在最终版本保留了预处理的全局 `fact` 数组查表法。

## 💻 究极融合版 C++ 代码

该代码兼顾了数学公式的可读性与 C++ STL 的工程优雅度，可以直接复制运行。

```cpp
#include <iostream>
#include <vector>
#include <numeric>   // 提供 std::iota
#include <algorithm> // 提供 std::find

using namespace std;
using ll = long long;

const int N = 20;
ll fact[N + 1];

// 预处理阶乘数组 (0! 到 20!)
// 20! 约为 2.43 * 10^18，在 64 位有符号整数 (long long) 的安全范围内
void precompute() {
    fact[0] = 1;
    for (int i = 1; i <= N; i++) {
        fact[i] = fact[i - 1] * i;
    }
}

void solve() {
    int op, n;
    cin >> op >> n;

    // 优雅地初始化候选数字集合 {1, 2, ..., n}
    vector<int> numbers(n);
    iota(numbers.begin(), numbers.end(), 1); 

    if (op == 1) {
        // --- 操作 1: 已知序号 k 求排列 (逆康托展开) ---
        ll k;
        cin >> k;
        k--; // 转换为 0-indexed 以便进行除法和取余运算

        for (int i = 0; i < n; i++) {
            ll f = fact[n - 1 - i]; // 直接查表获取当前位对应的“大类”排列数
            
            int idx = k / f; // 计算应当选取候选集中排第几的数字 (即下标)
            cout << numbers[idx] << (i == n - 1 ? "" : " ");
            
            numbers.erase(numbers.begin() + idx); // 从候选集中移除已用数字
            k %= f; // 更新剩余需要跳过的排列数
        }
        cout << "\n";
        
    } else if (op == 2) {
        // --- 操作 2: 已知排列求序号 k (康托展开) ---
        ll k = 1; // 官方技巧：直接初始化为 1-indexed，最终无需再做 +1 修正

        for (int i = 0; i < n; i++) {
            int x;
            cin >> x;
            
            ll f = fact[n - 1 - i]; // 获取当前位对应的“大类”排列数
            
            // 使用 find 寻找当前数字 x 在候选集中的位置(迭代器)
            auto it = find(numbers.begin(), numbers.end(), x);
            
            // 迭代器相减得到下标 idx，它完美等价于：“集合中有几个可用数字比 x 小”
            int idx = it - numbers.begin(); 
            
            k += idx * f; // 累加跳过的大类排列数
            numbers.erase(it); // 从候选集中移除处理过的数字
        }
        cout << k << "\n";
    }
}

int main() {
    // 优化 C++ 的输入输出流速度，防止在多组数据时超时
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    precompute();

    int t;
    if (cin >> t) {
        while (t--) {
            solve();
        }
    }
    return 0;
}
