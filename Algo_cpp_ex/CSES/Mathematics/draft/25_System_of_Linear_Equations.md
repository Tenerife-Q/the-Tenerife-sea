# CSES 25 - System of Linear Equations (模域上的线性方程组) 深度解析

本文将从线性代数的数学原理出发，推导有限域上的高斯消元法，并通过直观的矩阵可视化来详细解答**自由变量与特解的提取**过程，最后对比标准实现与官方实现的差异。

---

## 1. 线性代数数学原理与公式推导

### 1.1 模 $P$ 意义下的有限域
在实数系 $\mathbb{R}$ 中，解线性方程组可以使用加、减、乘、除。
本题中，所有的运算都在对 $P = 10^9+7$ 取模的意义下进行。因为 $P$ 是一个质数，在模 $P$ 意义下的整数 $0 \sim P-1$ 构成了一个**有限域 (Finite Field)**。这意味着我们在实数矩阵中的减法和除法，在这里可以通过**带模运算**和**求逆元**来实现，且消元过程不会破坏解的等价性。

- **减法**：$A - B \equiv (A - B \pmod P + P) \pmod P$
- **除法**：除以一个数 $X$，等价于乘以 $X$ 在模 $P$ 下的乘法逆元。
  根据**费马小定理**，如果 $\gcd(X, P) = 1$，则 $X^{P-1} \equiv 1 \pmod P$。
  两边同乘 $X^{-1}$ 得到 $X^{-1} \equiv X^{P-2} \pmod P$。
  这也是代码中 `pow(x, M - 2)` 这段求逆元代码的数学核心原理。

### 1.2 高斯-若尔当消元法 (Gauss-Jordan Elimination)
高斯消元的目标是通过三种初等行变换（不改变方程的解），将增广矩阵化为**简化行阶梯形矩阵 (Reduced Row Echelon Form, RREF)**：
1. 交换两行 (Row Swap)
2. 将一行的所有元素乘以非零常数 $k$ (Row Multiply)
3. 将一行的 $k$ 倍加到另一行 (Row Add)

由于我们要求解变量的具体值，我们的最终目标是把矩阵左侧系数化成一个类似“单位阵”的形式。

## 2. “特解”和“提取答案”的矩阵可视化讲解

你提到了**没有看懂如何提取答案和特解**，这一部分是高斯消元中最精妙的地方，也是存在**自由变量 (无穷多解)** 时的核心处理逻辑。

我们用一个存在**自由变量**的例子来进行矩阵可视化演示。
假设经过若干步消元后，我们的增广矩阵化简为了如下的最后形式（包含 3 个变量 $x_0, x_1, x_2$，共有 3 个方程）：

$$
\begin{bmatrix}
1 & 0 & 2 & | & 5 \\
0 & 1 & 3 & | & 4 \\
0 & 0 & 0 & | & 0
\end{bmatrix}
$$

### 2.1 如何解读这个矩阵？
将上面的增广矩阵还原成方程代数式：
1. 第 0 行：$1 \cdot x_0 + 0 \cdot x_1 + 2 \cdot x_2 = 5 \implies x_0 + 2x_2 = 5$
2. 第 1 行：$0 \cdot x_0 + 1 \cdot x_1 + 3 \cdot x_2 = 4 \implies x_1 + 3x_2 = 4$
3. 第 2 行：$0 \cdot x_0 + 0 \cdot x_1 + 0 \cdot x_2 = 0 \implies 0 = 0$ (恒等式，无约束力)

### 2.2 什么是主元变量？什么是自由变量？
- 观察第 0 列和第 1 列，它们在某一行有一个**领头的 1 (Pivot 1)** 并且该列其它位置全是 0。这说明 $x_0$ 和 $x_1$ 已经被成功“孤立”出来了。它们叫做**主元变量 (Pivot Variables)**。
- 观察第 2 列（对应 $x_2$），它**没有**自己领头的 1。也就是说，在最终方程中，没有哪个方程是呈“ $x_2 = \dots$ ”的独占形式。这叫**自由变量 (Free Variable)**。

自由变量的含义就是：它可以**任意取值**而不破坏方程组合法性！
但由于题目说明："You can print any valid solution" (只需打印任意一组有效解即可)。
既然 $x_2$ 可以随便取，我们作为一个懒惰的程序员，直接**赋予它最简单的特解：令全局自由变量全部等于 0！**

### 2.3 当令自由变量为 0 时发生的神奇现象
一旦我们指定 $x_2 = 0$，上面的方程组会立刻坍缩：
1. $x_0 + 2(0) = 5 \implies x_0 = 5$
2. $x_1 + 3(0) = 4 \implies x_1 = 4$

**结论：** 
只要把所有的自由变量强行设为 0，那么主元变量的值，**就等于增广矩阵最后一列（即等号右侧常数项）的值！**
这也叫求“特解”。这就解释了官方代码中提取答案的那段逻辑：

```cpp
for (int i = 0; i < m; i++) { // 遍历 x0 到 x_m-1 每一个变量
    bool found = false;
    for (int j = 0; j < n; j++) {
        // 如果发现第 j 行方程专门用来解第 i 个变量 (主元变量)
        if (chosen[j] == i) {
            cout << matrix[j][m] << " "; // 变量的值直接就是等式右侧的系数！
            found = true;
            break;
        }
    }
    // 如果没有任何一行拿第 i 个变量当主元，它就是一个自由变量！
    // 赋予特解 0
    if (!found) cout << 0 << " "; 
}
```

## 3. 标准版代码 VS 官方版代码的差异分析

你问到之前那份带有 `swap(a[r], a[pivot]);` 的代码（即课本标准版高斯消元）还有没有必要讲解。
答案是：**非常有必要，因为它们本质完全相同，只是在数据结构的管理上（排版上）有所不同！**

我们来看看它俩的区别。它代表了算法工程实现上的两种不同哲学。

### 3.1 标准版（Row Swap / 行交换形态）
标准版必须维护一个绝对的“阶梯”形状。它的流程是：
1. 针对第 $c$ 列，寻找底下哪一行在这个位置有非 0 元素。
2. 找到后，**通过 `swap` 将其强制置换到顶部的第 `r` 行**。
3. 这样做保证了主对角线上是一条漂亮的 `1` 斜线。

**优点**：数学形态上最规整，非常契合《线性代数》书上的证明和手工演算。处理无解或者有解的逻辑很清晰（查底部的 0 行等式右边是不是 0）。
**缺点**：在写代码时频繁操作整行的 `swap` 在某些大矩阵中（就算用了 vector 也会带来额外的理解成本）显得有些“强迫症”。

### 3.2 官方版（Chosen Mapping / 映射认领形态）
官方给的这份代码：**完全没有使用 `swap` 交换行！**
它的逻辑可以认为是处于一个“无序的房间”里。
1. 第 `i` 个方程，自己从左到右去扫描变量（列 `j`）。
2. 它发现当前 $x_j$ 的系数不为 0，它就喊一声：“**这个 $x_j$ 变量归我管了！**” (`chosen[i] = j`)。
3. 接着它把自己作为武器，去杀掉（消去）别的所有方程里的 $x_j$。
4. 下一个方程再去找下一个没被归一化的列认领，重复这个过程。

由于并没有因为认领的先后顺序去交换行，所以它在内存中可能长得并不像个阶梯。例如，可能第 1 行解出了 $x_2$，第 2 行解出了 $x_0$。
但正因为它用 `chosen` 数组记录了“哪一行等于哪个主元”，所以它不需要排好序再输出，最后输出只需查一遍 `chosen` 字典即可！

**优点：** 
舍弃了不必要的物理行交换，用 `chosen` 数组充当行和列的逻辑映射。代码更加简短轻快。
当处理矛盾时，只需查 `!found && matrix[i][m] != 0` （没有管得了的变量了，而右边还不是 0，则是引发了 $0 = C$ 的荒谬矛盾）。

### 4. 总结
不管是哪份代码，它们数学上严格等价：**将矩阵化为 RREF、剔除自由变量赋 0 得特解、发现常量矛盾即报无解**。官方题解版利用了 `chosen` 取代了 `行阶梯swap排序`，这是一种很优秀的工程技巧，非常值得背诵和领会。

# 👑 经典高斯-约旦消元法：变量、概念与矩阵推导

在开始矩阵推导之前，我们先明确代码中各个核心变量的含义，并复习三个最重要的线性代数概念。

### 1. 核心数组与指针标号的含义

| 变量/数组 | 标号含义与用处 |
| :--- | :--- |
| `n`, `m` | `n` 是方程个数（行数），`m` 是变量个数（列数）。 |
| `a[i][j]` | **增广矩阵 (Augmented Matrix)**。大小为 `n * (m+1)`。`a[i][m]` 永远存储等号右边的常数项 $b_i$。 |
| `c` | **当前列指针 (Column)**。代表我们正在处理第 $c$ 个变量 $x_c$。循环从 `0` 到 `m-1` 推进。 |
| `r` | **当前行指针 (Row)**。代表我们正在确定第 $r$ 个有效方程。**循环结束时，`r` 就是矩阵的“秩 (Rank)”**。 |
| `pivot` | 寻找主元时的临时行标。用来在第 $c$ 列寻找非 `0` 系数。 |
| `pivot_col` | **主元列记录表**。`pivot_col[i] = c` 表示：经过化简后，第 `i` 行的主元是第 `c` 列。用来在最后提取答案。 |
| `ans[m]` | 最终的答案数组，初始全为 `0`，用于存储特解。 |

### 2. 必须找回的线性代数记忆

*   **秩 (Rank, 即代码中的 `r`)**：表示方程组中**真正有用（线性无关）**的方程个数。如果两个方程是 `x+y=2` 和 `2x+2y=4`，其实它们是一个方程，秩只有 1。
*   **自由变量 (Free Variable)**：如果在推导过程中，发现某一个变量所在的列，找不到任何系数不为 0 的行（即无法作为主元），它就是自由变量。它的取值是**任意的**。
*   **无解与特解 (Inconsistent & Particular Solution)**：
    *   **无解**：化简后出现 `0 = C (C ≠ 0)` 的情况。
    *   **特解**：当有无穷多解（存在自由变量）时，题目要求输出**任意一组解**。最方便的特解就是**把所有自由变量全部强制设为 `0`**。

### 3. 🎬 矩阵演练：一步步带你跑通代码

> 💡 **特别说明**：为了人类的直观阅读，以下推导使用**普通十进制整数**进行演示。实际代码中，所有的乘法/除法都是在 **模 $10^9+7$** 的有限域下进行的（即使用 `inverse` 求逆元替代除法），但**这完全不影响矩阵变换的逻辑本质**。

假设我们有以下方程组（**3个方程，4个变量**，这是一个存在自由变量的绝佳例子）：
1.  $1x_0 + 2x_1 + 0x_2 + 1x_3 = 4$
2.  $2x_0 + 4x_1 + 1x_2 + 2x_3 = 11$
3.  $1x_0 + 2x_1 + 1x_2 + 1x_3 = 7$

初始化增广矩阵 `a`：
```text
  x0 x1 x2 x3 | b
[  1  2  0  1 | 4  ]  <-- 第 0 行
[  2  4  1  2 | 11 ]  <-- 第 1 行
[  1  2  1  1 | 7  ]  <-- 第 2 行
```
初始状态：`r = 0`。外层循环 `c` 开始从 `0` 到 `3` 遍历。

#### 🟢 Step 1: 处理第 `c = 0` 列 (变量 $x_0$)
*   **找主元**：当前 `r = 0`。从第 `0` 行往下找 `a[i][0] != 0` 的行。发现 `a[0][0] = 1`，主元就是第 0 行。
*   **交换**：`swap(a[0], a[0])` (自己和自己换)。
*   **化主元为 1**：已经是 1，无需操作。
*   **消去其他行**：
    *   消除第 1 行：`R1 = R1 - 2 * R0`
    *   消除第 2 行：`R2 = R2 - 1 * R0`
*   **矩阵变为**：
```text
  x0 x1 x2 x3 | b
[  1  2  0  1 | 4  ]  <-- 第 0 行搞定！
[  0  0  1  0 | 3  ]  
[  0  0  1  0 | 3  ]  
```
*   **记录与推进**：`pivot_col.push_back(0)`。**行指针 `r` 加一变成 1**。

#### 🟡 Step 2: 处理第 `c = 1` 列 (变量 $x_1$)
*   **找主元**：当前 `r = 1`。从第 `1` 行开始往下找，看第 1 列是否有非 0。
    发现 `a[1][1] = 0`, `a[2][1] = 0`。
*   **触发边界效应**：找不到主元！触发代码：`if (pivot == n) continue;`
*   **线代意义**：$x_1$ 这一列下面全是 0，说明无法用剩下的方程确定 $x_1$。**$x_1$ 是自由变量！**
*   **状态**：矩阵不变化，**行指针 `r` 依然是 1，不推进！** (因为我们没消耗有用的方程)。

#### 🟢 Step 3: 处理第 `c = 2` 列 (变量 $x_2$)
*   **找主元**：当前 `r = 1`。从第 `1` 行往下找第 2 列。发现 `a[1][2] = 1`。
*   **交换**：`swap(a[1], a[1])`。
*   **化主元为 1**：已经是 1。
*   **消去其他行 (包含上方！)**：
    *   上方第 0 行：`a[0][2]` 本来就是 0，无需消元。
    *   下方第 2 行：`R2 = R2 - R1`
*   **矩阵变为**：
```text
  x0 x1 x2 x3 | b
[  1  2  0  1 | 4  ]
[  0  0  1  0 | 3  ]  <-- 第 1 行搞定！
[  0  0  0  0 | 0  ]  <-- R2 被消成全 0，说明它是冗余方程
```
*   **记录与推进**：`pivot_col.push_back(2)`。**行指针 `r` 加一变成 2**。

#### 🟡 Step 4: 处理第 `c = 3` 列 (变量 $x_3$)
*   **找主元**：当前 `r = 2`。看第 `2` 行及以下的第 3 列，全为 0。
*   **触发边界效应**：`if (pivot == n) continue;`
*   **线代意义**：**$x_3$ 也是自由变量！**
*   **状态**：结束消元。最终 **秩 `r = 2`**。

### 4. 🚀 结案：无解判定与提取特解

代码的最后两段逻辑是整个算法的精髓。经过上面的消元，我们得到了一个**行最简阶梯型矩阵 (RREF)**：

```text
  x0 x1 x2 x3 | b
[  1  2  0  1 | 4  ]   (方程记录在 pivot_col[0] = 0)
[  0  0  1  0 | 3  ]   (方程记录在 pivot_col[1] = 2)
[  0  0  0  0 | 0  ]   (无效冗余行)
```

#### 🚫 第一步：检查无解
```cpp
for (int i = r; i < n; ++i) {
    if (a[i][m] != 0) {
        cout << -1 << "\n"; return 0;
    }
}
```
因为矩阵真正的秩只有 `r=2`，说明第 2 行及以后的所有左边变量系数全是 `0`。
我们只需要检查右边的常数项 `a[i][m]` 是不是也是 `0`。
*   如果是 `0`（就像我们上面一样 `[0 0 0 0 | 0]`），说明是 `0 = 0`，合理。
*   如果常数项不是 `0`（例如 `[0 0 0 0 | 5]`），那是 `0 = 5` 的致命矛盾，代表方程组互相冲突，**直接输出 `-1` 判定无解**。

#### 🎯 第二步：提取特解 (处理自由变量)
方程组现在的实际数学表达是：
1.  $x_0 + 2x_1 + x_3 = 4$
2.  $x_2 = 3$

怎么求一组合法的 `x0, x1, x2, x3`？
代码用了一个极其聪明的操作：

```cpp
vector<long long> ans(m, 0); 
```
**一上来，不管三七二十一，把所有的变量先假设为 `0`！**
这意味着我们的自由变量 $x_1$ 和 $x_3$ 已经被我们**主动赋值为 `0`** 了。
那么上面的方程瞬间坍塌为：
1.  $x_0 + 0 + 0 = 4 \implies x_0 = 4$
2.  $x_2 = 3 \implies x_2 = 3$

接下来怎么让代码自动做到这一点呢？看这个循环：
```cpp
for (int i = 0; i < r; ++i) {
    ans[pivot_col[i]] = a[i][m];
}
```
*   `i = 0`: `pivot_col[0]` 是 `0`。所以 `ans[0] = a[0][4]` 也就是 `ans[0] = 4`。
*   `i = 1`: `pivot_col[1]` 是 `2`。所以 `ans[2] = a[1][4]` 也就是 `ans[2] = 3`。

最终得到的 `ans` 数组为 `[4, 0, 3, 0]`。
完美避开了复杂的回代计算（Back-substitution），直接提取常数项就拿到了答案！这只有在使用高斯-约旦消元法（把主元上下都消成 0）时才能做到。
