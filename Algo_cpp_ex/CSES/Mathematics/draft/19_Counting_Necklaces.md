# Counting Necklaces (项链计数问题) - 核心原理解析与完整证明

这道题是组合数学与群论（Burnside 引理 / Pólya 定理）的经典应用。
抛开高深晦涩的群论名词，本文将用最直观的**“画表格算两次”**和**“操场跑圈”**的日常逻辑，为你详细证明并推导出最终的计算公式。

---

## 一、 问题背景与难点

**题目大意**：
用 $m$ 种颜色的珠子，串成一条长度为 $n$ 的首尾相连的项链。由于项链是可以**旋转**的（转动后看起来一样的视为同一种），求本质不同的项链总数。结果对 $10^9+7$ 取模。（$1 \le n, m \le 10^6$）

**为什么不能直接计算？**
如果是直线排列，方案数显然是 $m^n$。但连成项链后，不同的方案在旋转后会发生重复。
且**每种项链重复的次数是不一样的**：
- `红-蓝-红-蓝` 转 2 次就和自己重合了，它只有 2 个不同形态。
- `红-红-红-红` 怎么转都一样，它只有 1 个形态。

正因为每种情况“变出的花样”数量不同，所以**不能直接用 $m^n$ 除以 $n$**。我们需要换一种极其聪明的数数方法。

---

## 二、 核心证明 1：为什么要“算保持不变的数量，然后求平均值”？

既然不能直接数项链，数学家（Burnside）想出了一个绝妙的逆向思维：**去数“旋转操作”，看看在每次旋转下，有多少种直线排列方案是“长得一模一样”的，然后求个平均值。**

> **结论：本质不同的项链总数 = 所有旋转操作下【保持不变的方案数】的总和 $\div$ 旋转操作的总数 $n$。**

### 详细证明过程（画表格“算两次”法）

我们用数学中非常强大的思想：**算两次 (Double Counting)**。
假设 $n=4$ 颗珠子，我们把所有的直线排列方案（共 $m^4$ 种）和 4 种旋转操作（转0、1、2、3步）画成一张**巨大的二维表格**。

- **填表规则**：如果某个直线方案在经历某种旋转后，**长得和原来一模一样（保持不变）**，我们就在格子里填 `1`；如果变了，就填 `0`。

我们拿几条具有代表性的项链来举例展示这张表的部分内容：
1. **项链 A（4种形态）**：如 `红-蓝-绿-黄`，每次转都不一样，有 4 个不同的直线方案属于这个“家族”。
2. **项链 B（2种形态）**：如 `红-蓝-红-蓝`，转2步后和自己重合，只有 2 个不同的直线方案属于这个家族。
3. **项链 C（1种形态）**：如 `红-红-红-红`，怎么转都一样，只有 1 个直线方案。

| 旋转步数 \ 具体的直线方案 | ... | A形1 | A形2 | A形3 | A形4 | ... | B形1 | B形2 | ... | C形1 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **转 0 步 (不转)** | ... | 1 | 1 | 1 | 1 | ... | 1 | 1 | ... | 1 |
| **转 1 步** | ... | 0 | 0 | 0 | 0 | ... | 0 | 0 | ... | 1 |
| **转 2 步** | ... | 0 | 0 | 0 | 0 | ... | 1 | 1 | ... | 1 |
| **转 3 步** | ... | 0 | 0 | 0 | 0 | ... | 0 | 0 | ... | 1 |

现在，我们想知道**整个表格里一共有多少个 `1`**。有两种数法：

#### 1. 按行数（横向看）
把每一行里的 `1` 加起来，这就等于：**“在每种旋转操作下，保持不变的方案数总和”**。

#### 2. 按列数（纵向看，见证奇迹的时刻）
我们把属于**“同一条真实项链（家族）”**的列打包在一起看：
- **项链 A 家族**：有 4 种形态，占了 4 列。因为每种形态只在转 0 步时和自己一样，所以每列只有 1 个 `1`。整个家族贡献的 `1` 的数量为：$4 \times 1 = \mathbf{4}$。
- **项链 B 家族**：有 2 种形态，占了 2 列。因为每种形态在转 0 步和转 2 步时和自己一样，所以每列有 2 个 `1`。整个家族贡献的 `1` 的数量为：$2 \times 2 = \mathbf{4}$。
- **项链 C 家族**：有 1 种形态，占了 1 列。因为它在 4 种旋转下都和自己一样，所以这 1 列有 4 个 `1`。整个家族贡献的 `1` 的数量为：$1 \times 4 = \mathbf{4}$。

**极其震撼的结论出现了：**
无论一条项链长什么样，变出了几种形态，**每一条“本质不同”的项链，它整个家族在表格里贡献的 `1` 的总数，永远刚好是 $n$ 个（在这个例子里是 4）！**

既然每一条真实项链都刚好贡献了 $n$ 个 `1`，那么：
$$ \text{表格里 1 的总数} = \text{本质不同的项链总数} \times n $$

我们将“按行数”和“按列数”的结果画上等号：
$$ \text{每种旋转保持不变的方案数总和} = \text{本质不同的项链总数} \times n $$
把 $n$ 移项除过去：
$$ \text{本质不同的项链总数} = \frac{\text{每种旋转保持不变的方案数总和}}{n} $$
**(证明完毕！这就是 Burnside 引理的最直观理解。)**

## 三、 核心证明 2：怎么算“保持不变”的数量？

现在任务变成了：把一条长度为 $n$ 的项链旋转 $k$ 步（$0 \le k < n$），它如果要“长得一模一样”，必须要满足什么条件？
- 答案是：原来在位置 $x$ 的珠子，必须和转过去后的位置 $(x+k) \pmod n$ 的珠子**颜色相同**。
- 这会导致珠子形成若干个**“必须同色的循环分组”**。每个分组可以独立从 $m$ 种颜色中任选 1 种。

> **结论：转 $k$ 步产生的分组数量，恰好等于 $\gcd(n, k)$（$n$ 和 $k$ 的最大公约数）。**

### 详细证明过程（“操场跑圈”法）

假设操场上有 $n$ 个等距离的标记点（编号 $0$ 到 $n-1$）。
你从某个点起跳，每次**跨越 $k$ 个间距**。因为是绕圈跳，一直跳下去，你最终一定会回到起点，这就形成了一个“闭环的分组”。

1. **总共跨越了多远？**
   假设你跳了 $L$ 次刚好**第一次**回到起点。这意味着你跨越的总距离 $L \times k$ 刚好绕了操场整整几圈。
   也就是说，$L \times k$ 必须是 $n$ 的倍数。
   既然 $L \times k$ 既是 $k$ 的倍数也是 $n$ 的倍数，且要是“第一次”回到起点，那么总距离必定是 $n$ 和 $k$ 的**最小公倍数 $\text{lcm}(n, k)$**。

2. **这个闭环分组里有几个点？**
   跳的次数 $L$，就是这个分组里的点数：
   $$ L = \frac{\text{lcm}(n, k)}{k} $$
   根据小学数学公式：两个数的乘积等于它们的最大公约数乘最小公倍数，即 $\text{lcm}(n, k) = \frac{n \times k}{\gcd(n, k)}$。
   代入上式得到：
   $$ L = \frac{\frac{n \times k}{\gcd(n, k)}}{k} = \frac{n}{\gcd(n, k)} $$
   **这意味着，任何一个分组里，都刚好有 $\frac{n}{\gcd(n, k)}$ 颗珠子。**

3. **一共分成了几个组？**
   既然每个组的大小都是一样的，那一共有几个组呢？
   $$ \text{分组数量} = \frac{\text{总珠子数 } n}{\text{每个分组的珠子数 } \frac{n}{\gcd(n, k)}} = \gcd(n, k) $$
**(证明完毕！)**

既然分成了 $\gcd(n, k)$ 个独立的组，每个组内部必须同色（从 $m$ 种颜色中选 1 种），所以转 $k$ 步保持不变的染色方案数就是：
$$ m^{\gcd(n, k)} $$

结合上面的表格法，不同项链的总数公式就呼之欲出了：
$$ \text{Ans} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)} $$

## 四、 算法优化：欧拉函数 $\phi$ 的引入

虽然得出了公式，但题目数据范围 $n \le 10^6$。如果直接写个 `for` 循环把这 $n$ 项加起来，每次还要求 $\gcd$，可能会超时。我们需要打包优化。

我们知道 $\gcd(n, k)$ 的结果一定是 $n$ 的某个**约数**。
假设这个约数是 $d$。我们需要知道，在 $0 \le k < n$ 范围里，**有几个 $k$ 会让 $\gcd(n, k) = d$**？

- $\gcd(n, k) = d$ 等价于 $\gcd(\frac{n}{d}, \frac{k}{d}) = 1$。
- 换句话说，我们要找的是小于 $\frac{n}{d}$ 并且和 $\frac{n}{d}$ **互质**的整数 $\frac{k}{d}$ 的个数。
- 在数学里，这就是**欧拉函数 $\phi(\frac{n}{d})$** 的定义！

于是，我们不需要遍历一百万次 $k$，只需要遍历 $n$ 的所有约数 $d$（$10^6$ 以内的数最多只有 240 个约数）：
$$ \text{Ans} = \frac{1}{n} \sum_{d|n} \phi\left(\frac{n}{d}\right) m^d $$
这个公式将计算次数从 $O(N)$ 暴降到 $O(\sqrt{N})$，实现降维打击。

## 五、 最终 C++ 代码实现

*注：题目要求结果对 $10^9+7$ 取模。根据费马小定理，公式最外层的“除以 $n$”，需要转化为“乘以 $n$ 在模 $10^9+7$ 下的乘法逆元”，即 $n^{MOD-2} \pmod{MOD}$。*

```cpp
#include <iostream>

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂算法：计算 (base^exp) % MOD
long long power(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

// 求正整数 x 的欧拉函数 Φ(x)
long long get_phi(long long x) {
    long long res = x;
    for (long long i = 2; i * i <= x; i++) {
        if (x % i == 0) {
            // 将质因子 i 除干净
            while (x % i == 0) x /= i;
            // 欧拉函数公式推导：res = res * (1 - 1/i)
            res -= res / i;
        }
    }
    if (x > 1) res -= res / x;
    return res;
}

int main() {
    // 优化标准输入输出流，加快执行速度
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n, m;
    if (!(cin >> n >> m)) return 0;

    long long ans = 0;
    
    // 枚举 n 的所有约数 d (只需枚举到 sqrt(n))
    for (long long i = 1; i * i <= n; i++) {
        if (n % i == 0) {
            long long d1 = i;
            // 累加 d1 对应的部分：Φ(n/d1) * m^d1
            ans = (ans + get_phi(n / d1) % MOD * power(m, d1)) % MOD;
            
            long long d2 = n / i;
            // 如果 d1 和 d2 不相等（避免完全平方数时重复计算）
            if (d1 != d2) {
                // 累加 d2 对应的部分：Φ(n/d2) * m^d2
                ans = (ans + get_phi(n / d2) % MOD * power(m, d2)) % MOD;
            }
        }
    }

    // 最后除以 n，等价于乘以 n 的逆元 (n^(MOD-2) % MOD)
    ans = (ans * power(n, MOD - 2)) % MOD;
    
    cout << ans << "\n";
    
    return 0;
}
