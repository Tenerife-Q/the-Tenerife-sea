# 算法解析：Counting Necklaces (项链计数问题)

这道题是组合数学与群论中极其经典的问题。我们将把实际问题抽象为数学定理，并用最直观的**“画表格算两次”**和**“操场跑圈”**的日常逻辑，为你详细推导并证明最终的计算公式。

## 一、 将实际问题抽象为数学问题

**实际问题**：
用 $m$ 种颜色的珠子，串成一条长度为 $n$ 的首尾相连的项链。由于项链是可以**旋转**的（转动后看起来一样的视为同一种），求本质不同的项链总数。结果对 $10^9+7$ 取模（$1 \le n, m \le 10^6$）。

**抽象难点**：
如果是一条固定的直线，排列方案数显然是 $m^n$。但连成项链后，不同的方案在旋转后会发生重复。
且**每种项链重复的形态数量是不一样的**：
- `红-蓝-红-蓝` 转 2 步就和自己重合了，它只有 2 个不同形态。
- `红-红-红-红` 怎么转都一样，它只有 1 个形态。

正因为每种情况“变出的花样”数量不同，所以**不能简单地用 $m^n$ 除以 $n$**。我们需要引入群论中的强大工具：**Burnside 引理（Burnside's Lemma）**。

---

## 二、 定理的本质与两大核心问题

Burnside 引理提供了一种绝妙的逆向思维：**不去数项链，去数“旋转操作”**。我们观察在每次旋转操作下，有多少种直线排列方案是“长得一模一样”（即在该操作下保持不变）的，然后求个平均值。


> **核心定理（Burnside 引理通俗版）**：
> 本质不同的项链总数 = 把所有旋转操作下【保持不变的方案数】加起来 $\div$ 旋转操作的总数 $n$。

理解这个定理，只需要彻底搞懂以下两个核心问题：
1. **为什么对“保持不变的数量”求平均值，刚好就是真实的不同项链数？**
2. **在每次旋转下，怎么计算“保持不变”的数量？**

---

## 三、 核心问题 1：画表格证明“求平均值”的本质

我们用数学中非常强大的思想：**算两次 (Double Counting)** 来证明为什么求平均值是对的。这其实是群论中**轨道-稳定子定理（Orbit-Stabilizer Theorem）**的具象化表达。

假设 $n=4$ 颗珠子，我们将所有的直线排列方案和 4 种旋转操作（转 0、1、2、3 步）画成一张二维表格。
- **填表规则**：如果某个直线方案在经历某种旋转后，**长得和原来一模一样**，在格子里填 `1`；如果变了，填 `0`。

我们拿几条代表性的项链举例：
- **A 家族**：有 4 种形态，每种形态只在转 0 步时不变。
- **B 家族**：有 2 种形态，每种形态在转 0 步和转 2 步时不变。
- **C 家族**：有 1 种形态（全同色），在 4 种旋转下都不变。

| 旋转步数 \ 直线方案 | A形1 | A形2 | A形3 | A形4 | B形1 | B形2 | C形1 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **转 0 步 (不转)** | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
| **转 1 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |
| **转 2 步** | 0 | 0 | 0 | 0 | 1 | 1 | 1 |
| **转 3 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |

现在，我们想知道**整个表格里一共有多少个 `1`**。有两种数法：

### 方法 A：按行数（横向看）
把每一行里的 `1` 加起来，这就等于：**“在每种旋转操作下，保持不变的方案数总和”**。

### 方法 B：按列数（纵向看，本质规律的展现）
我们把属于**“同一条真实项链（同一个轨道 Orbit）”**的列打包在一起看：
- **项链 A 家族**：有 4 种形态（轨道大小为 4）。因为每种形态只在转 0 步时和自己一样（稳定子大小为 1），整个家族贡献的 `1` 数量为：$4 \times 1 = \mathbf{4}$。
- **项链 B 家族**：有 2 种形态（轨道大小为 2）。因为每种形态在转 0 步和转 2 步时和自己一样（稳定子大小为 2），整个家族贡献的 `1` 数量为：$2 \times 2 = \mathbf{4}$。
- **项链 C 家族**：有 1 种形态（轨道大小为 1）。在 4 种旋转下都和自己一样（稳定子大小为 4），整个家族贡献的 `1` 数量为：$1 \times 4 = \mathbf{4}$。

**震撼的结论：**
无论一条项链长什么样，**每一条“本质不同”的独立项链，它整个家族在表格里贡献的 `1` 的总数，永远刚好是 $n$ 个！**（因为 `轨道大小 × 稳定子大小 = 群的大小 n`，这就是 Orbit-Stabilizer Theorem 的精髓）。

既然每一条真实项链都刚好贡献了 $n$ 个 `1`，那么：

$$
\text{表格里 1 的总数} = \text{本质不同的项链总数} \times n
$$

将“按行数”和“按列数”的结果画上等号，并把 $n$ 移项除过去：

$$
\text{本质不同的项链总数} = \frac{\text{每种旋转保持不变的方案数总和}}{n}
$$

**这就解释了为什么是求平均值。**

## 四、 核心问题 2：操场跑圈证明 $\gcd(n, k)$

现在任务变成了：把一条长度为 $n$ 的项链旋转 $k$ 步，如果它要“长得一模一样”，必须要满足什么条件？

答案是：原来在位置 $x$ 的珠子，必须和位置 $(x+k) \pmod n$ 的珠子**颜色相同**。
这会导致珠子形成若干个**“必须同色的闭环分组（循环节 Cycle）”**。每个分组可以独立从 $m$ 种颜色中任选 1 种。


> **结论：转 $k$ 步产生的分组数量，恰好等于 $\gcd(n, k)$（最大公约数）。**

### 详细推导（循环群生成元视角）

假设操场上有 $n$ 个点，编号 $0$ 到 $n-1$，你每次跨越 $k$ 个间距。一直跳，直到第一次回到起点，就形成了一个“闭环的分组”。

1. **总共跨越了多远？**
   假设你跳了 $L$ 次刚好**第一次**回到起点，意味着总跨越距离 $L \times k$ 刚好是 $n$ 的倍数。
   因此，这个最短的总距离必定是 $n$ 和 $k$ 的**最小公倍数** $\operatorname{lcm}(n, k)$。

2. **这个闭环分组里有几个点？**
   跳的次数 $L$，就是这个分组里的点数（即轨道长度）：
   $$
   L = \frac{\operatorname{lcm}(n, k)}{k}
   $$
   根据数学公式 $\operatorname{lcm}(n, k) = \frac{n \times k}{\gcd(n, k)}$，代入上式得到：
   $$
   L = \frac{\frac{n \times k}{\gcd(n, k)}}{k} = \frac{n}{\gcd(n, k)}
   $$

3. **一共分成了几个组（循环节数量）？**
   总共有 $n$ 颗珠子，每个组内部有 $L$ 颗珠子，那么组数就是：
   $$
   \text{循环节数量} = \frac{n}{L} = \frac{n}{\frac{n}{\gcd(n, k)}} = \gcd(n, k)
   $$

既然分成了 $\gcd(n, k)$ 个独立的组，每个组内部必须同色（从 $m$ 种颜色中任选 1 种），所以转 $k$ 步保持不变的方案数就是：

$$
m^{\gcd(n, k)}
$$

结合表格法的平均值定理，得出基础通解公式（Pólya 计数定理的特例）：

$$
\text{Ans} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)}
$$

## 五、 结合欧拉函数的终极数论优化

基础公式需要计算 $n$ 次，但题目数据范围 $n \le 10^6$，如果每次都计算 $\gcd(n, k)$ 必定超时（复杂度 $O(N \log N)$）。
我们可以引入数论进行优化。观察到 $\gcd(n, k)$ 的结果一定是 $n$ 的**约数**。我们可以把 $\gcd$ 结果相同的 $k$ **打包合并计算**。

假设 $d$ 是 $n$ 的一个约数。在 $0 \le k < n$ 范围里，有几个 $k$ 会让 $\gcd(n, k) = d$？

条件 $\gcd(n, k) = d$ 等价于：
$$
\gcd\left(\frac{n}{d}, \frac{k}{d}\right) = 1
$$
换句话说，我们要找的是满足 $0 \le \frac{k}{d} < \frac{n}{d}$ 并且和 $\frac{n}{d}$ **互质**的整数 $\frac{k}{d}$ 的个数。
这恰好就是**欧拉函数 $\phi\left(\frac{n}{d}\right)$** 的数学定义！

于是，我们将原来按 $k$ 遍历，改为按 $n$ 的约数 $d$ 进行枚举遍历，得出终极优化公式：

$$
\text{Ans} = \frac{1}{n} \sum_{d \mid n} \phi\left(\frac{n}{d}\right) m^d
$$

在这个公式下，我们只需要遍历 $n$ 的约数（$10^6$ 以内的数最多只有 240 个约数），时间复杂度从线性的 $O(N)$ 暴降到了 $O(\sqrt{N})$，算法性能得到极大飞跃。

## 六、 最终 C++ 代码实现

*注：题目要求对 $10^9+7$ 取模。根据费马小定理（Fermat's Little Theorem），在模一个素数 $p$ 的意义下，除以 $n$ 等价于乘以 $n$ 的模反元素（乘法逆元），即 $n^{p-2} \pmod p$。在我们的代码中，由 `power(n, MOD - 2)` 实现。*

```cpp
#include <iostream>

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂算法：计算 (base^exp) % MOD，复杂度 O(log exp)
long long power(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

// 求正整数 x 的欧拉函数 Φ(x) 
// 即计算小于等于 x 且与 x 互质的正整数个数，复杂度 O(sqrt(x))
long long get_phi(long long x) {
    long long res = x;
    for (long long i = 2; i * i <= x; i++) {
        if (x % i == 0) {
            // 将质因子 i 除干净
            while (x % i == 0) x /= i;
            // 欧拉函数公式：res = res * (1 - 1/i)
            res -= res / i;
        }
    }
    // 如果最后剩余一个大于 1 的质因子（说明 x 本身包含一个比 sqrt(x) 大的素因子）
    if (x > 1) res -= res / x;
    return res;
}

int main() {
    // 优化标准输入输出流，加快执行速度，适合 ICPC 竞赛
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n, m;
    // 不断读入直到文件结束
    if (!(cin >> n >> m)) return 0;

    long long ans = 0;
    
    // 核心优化：枚举 n 的所有约数 d (只需枚举到 sqrt(n))
    for (long long i = 1; i * i <= n; i++) {
        if (n % i == 0) {
            // 情况 1: 约数 d1 = i
            long long d1 = i;
            // 累加 d1 对应的部分：Φ(n/d1) * m^d1
            ans = (ans + get_phi(n / d1) % MOD * power(m, d1)) % MOD;
            
            // 情况 2: 约数 d2 = n / i
            long long d2 = n / i;
            // 如果 d1 和 d2 不相等（避免 n 是完全平方数时，中间的约数重复计算）
            if (d1 != d2) {
                // 累加 d2 对应的部分：Φ(n/d2) * m^d2
                ans = (ans + get_phi(n / d2) % MOD * power(m, d2)) % MOD;
            }
        }
    }

    // 最后除以 n，在模 MOD 意义下等价于乘以 n 的乘法逆元 (n^(MOD-2) % MOD)
    ans = (ans * power(n, MOD - 2)) % MOD;
    
    cout << ans << "\n";
    
    return 0;
}
```

# 算法解析：Counting Necklaces（项链计数问题）的严谨数学推导

本文抛弃所有比喻，仅针对**项链**、**珠子**、**旋转**和**颜色序列**本身，通过逻辑推导说明 Burnside 引理在项链计数中的本质应用。



### 1. 什么是“一条真正的项链”？
在物理意义上，项链是一个**圆环**。但在计算时，我们首先处理的是长度为 $n$ 的**“直线颜色序列”**（总数为 $m^n$ 种）。

所谓“一条真正的项链”，在数学定义上是一个**等价类（Equivalent Class）**：即可以通过旋转互相转换的若干个直线序列的集合。

**示例（假设 $n=4$ 颗珠子，$m=2$ 种颜色）：**
* **项链 A**：包含 `{红, 蓝, 绿, 黄}`、`{黄, 红, 蓝, 绿}`、`{绿, 黄, 红, 蓝}`、`{蓝, 绿, 黄, 红}` 这 **4** 个直线序列。在圆环上转动，能呈现这 4 种顺序。
* **项链 B**：包含 `{红, 蓝, 红, 蓝}`、`{蓝, 红, 蓝, 红}` 这 **2** 个直线序列。无论怎么转，只能呈现这 2 种顺序。
* **项链 C**：包含 `{红, 红, 红, 红}` 这 **1** 个直线序列。任何旋转下顺序都不变。

**目标**：求出这些独立集合（等价类）的总数。由于每个集合包含的序列数量不相等（有的包含 $n$ 个，有的包含 $n$ 的约数个），因此不能直接用 $m^n / n$ 计算。

---

### 2. 为什么“保持不变的序列总数 $\div n$”就是项链总数？

我们统计所有 $m^n$ 个序列在 $n$ 种旋转操作（转 $0, 1, \dots, n-1$ 步）下的表现：如果旋转后序列与原序列完全一致，计为一次“不变”。



**每一条项链（集合）对“不变次数”总和的贡献：**

* **项链 A（包含 4 个序列）：** 每个序列仅在“转 0 步”时保持不变。
    * 贡献：$4 (\text{序列数}) \times 1 (\text{不变次数}) = \mathbf{4}$。
* **项链 B（包含 2 个序列）：** 每个序列在“转 0 步”和“转 2 步”时保持不变。
    * 贡献：$2 (\text{序列数}) \times 2 (\text{不变次数}) = \mathbf{4}$。
* **项链 C（包含 1 个序列）：** 这个序列在“转 0, 1, 2, 3 步”时全部保持不变。
    * 贡献：$1 (\text{序列数}) \times 4 (\text{不变次数}) = \mathbf{4}$。

**本质规律（轨道-稳定子定理）：**
无论一条项链包含多少个序列，**属于同一条项链的所有直线序列，在所有 $n$ 种旋转中产生的“不变次数”之和，永远精确等于 $n$。**

因此：
$$\text{本质不同的项链总数} = \frac{\sum_{k=0}^{n-1} (\text{在旋转 } k \text{ 步下保持不变的序列数})}{n}$$

---

### 3. 旋转 $k$ 步时，如何计算“保持不变”的序列数？

若一个序列在循环右移 $k$ 位后保持不变，则必须满足：**位置 $x$ 的颜色 = 位置 $(x+k) \pmod n$ 的颜色。**

这将导致珠子之间产生“颜色绑定”，形成若干个互不相交的**同色组（循环节）**：
* 绑定链条：$x \to (x+k) \to (x+2k) \to \dots \to x$。
* 链条回到起点 $x$ 时，移动的总步数必须既是 $k$ 的倍数，也是 $n$ 的倍数。
* 第一次回到起点时，总步数为 $n$ 和 $k$ 的**最小公倍数** $\operatorname{lcm}(n, k)$。



**推导组数（循环节数量）：**
1.  **单组珠子数**：每次移动 $k$ 步增加一颗珠子，故每组包含 $\frac{\operatorname{lcm}(n, k)}{k}$ 颗珠子。
2.  根据恒等式 $\operatorname{lcm}(n, k) = \frac{n \times k}{\gcd(n, k)}$，代入得：
    $$\text{单组珠子数} = \frac{n}{\gcd(n, k)}$$
3.  **总组数**：
    $$\text{组数} = \frac{n}{\text{单组珠子数}} = \frac{n}{n / \gcd(n, k)} = \gcd(n, k)$$

**结论：**
在旋转 $k$ 步的操作下，序列被划分为 $\gcd(n, k)$ 个独立的颜色组。每个组可以任选 $m$ 种颜色中的一种，故保持不变的方案数为：
$$\text{Count}(k) = m^{\gcd(n, k)}$$

---

### 4. 总结与最终公式

1.  **单次旋转贡献**：旋转 $k$ 步时，保持不变的序列数量为 $m^{\gcd(n, k)}$。
2.  **全集求和**：遍历所有可能的旋转步数 $k \in \{0, 1, \dots, n-1\}$。
3.  **求均值**：将总和除以旋转操作的总数 $n$。

**最终公式：**
$$\text{Ans} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)}$$

该公式可以通过欧拉函数进一步优化至 $O(\sqrt{N})$，但在理解层面，上述推导已涵盖了该问题的数学核心。

# 算法解析：Counting Necklaces 的基础与数论优化

本文档针对“项链计数”问题（Burnside 引理的具体应用），对比基础的直接计算法与基于欧拉函数的数论优化法。核心在于展示如何通过严格的代数推导，将算法的时间复杂度从线性的 $O(N \log N)$ 降维至 $O(\sqrt{N})$。

## 一、 基础解法：公式的直接映射 ($O(N \log N)$)

根据 Burnside 引理，长度为 $n$、颜色数为 $m$ 的项链，在只考虑旋转同构的情况下，本质不同的项链总数为：

$$\text{Ans} = \frac{1}{n} \sum_{i=1}^{n} m^{\gcd(n, i)} \pmod M$$

#### 细微区别 A：范围是 `1 到 n` 还是 `0 到 n-1`？
- 我们推导时，旋转步数是 `0 到 n-1` 步。
- 标答代码写的是 `i = 1 到 n` 步。
- **为什么两者一样？**
  因为在项链上，**旋转 0 步** 和 **旋转 n 步** 是一模一样的（刚好转回原位）。
  并且在求最大公约数时：$\gcd(n, n) = n$，而 $\gcd(0, n) = n$。两者算出来的次方数完全相同！标答写 `1 到 n` 主要是为了符合日常写 `for` 循环的习惯，并且自带的（或手写的） `gcd` 函数处理正整数更自然。

### 基础版 C++ 代码
此版本直接平移数学公式，逻辑极其清晰。在数据范围 $n \le 10^6$ 的竞赛题中，该 $O(N \log N)$ 复杂度的实现足以通过。

```cpp
#include <iostream>

using namespace std;
using ll = long long;

const int M = 1000000007;

int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}

int power(int a, int b) {
    if (b == 0) return 1;
    ll p = power(a, b / 2);
    p = p * p % M;
    if (b % 2 == 1) p = p * a % M;
    return p;
}

// 费马小定理求逆元
int inv(int x) {
    return power(x, M - 2);
}

int main() {
    int n, m;
    cin >> n >> m;

    ll sum = 0;
    // 直接遍历所有的旋转步数
    for (int i = 1; i <= n; i++) {
        sum += power(m, gcd(i, n));
        sum %= M;
    }
    
    // 结果乘以 n 在模 M 意义下的乘法逆元
    ll answer = sum * inv(n) % M;
    cout << answer << "\n";
    
    return 0;
}
```

## 二、 进阶解法：基于欧拉函数的数论优化 ($O(\sqrt{N})$)

当数据范围扩大（例如 $n \le 10^9$）时，基础解法会因大量的重复计算而超时。优化的核心在于**利用最大公约数的性质进行分组打包计算**。

### 1. 发现重复计算
在遍历 $i$ 时，$\gcd(n, i)$ 的结果必定属于 $n$ 的约数集合。大量不同的 $i$ 产生了相同的 $\gcd$ 结果，导致 $m^{\gcd(n, i)}$ 被重复计算。

举个具体的例子：假设 $n = 12$。
$i$ 从 1 到 12 遍历，$\gcd(12, i)$ 的结果分别是：
- $i=1 \Rightarrow \gcd=1$
- $i=2 \Rightarrow \gcd=2$
- $i=3 \Rightarrow \gcd=3$
- $i=4 \Rightarrow \gcd=4$
- $i=5 \Rightarrow \gcd=1$
- $i=6 \Rightarrow \gcd=6$
- $i=7 \Rightarrow \gcd=1$
- $i=8 \Rightarrow \gcd=4$
- $i=9 \Rightarrow \gcd=3$
- $i=10 \Rightarrow \gcd=2$
- $i=11 \Rightarrow \gcd=1$
- $i=12 \Rightarrow \gcd=12$

你看，虽然我们算了 12 次，但算出来的结果全都是 12 的约数（1, 2, 3, 4, 6, 12）。
**既然有很多 $\gcd$ 的结果是相同的，我们为什么要一个一个去加呢？**

### 2. 转换视角：从“遍历 $i$” 到 “遍历约数 $d$”

与其遍历所有的 $i$，我们不如直接把 $n$ 的所有约数找出来。
假设 $d$ 是 $n$ 的一个约数。我们只需要解决一个核心的数学问题：
> **在 $1$ 到 $n$ 这 $n$ 个数里面，到底有几个 $i$，满足 $\gcd(n, i) = d$？**

只要算出了这个“个数”，我们就可以直接用乘法**打包计算**：
$$\text{个数} \times m^d$$
把所有约数 $d$ 对应的这个结果加起来，就等价于原来那个长长的 $\sum$！

### 3. 严格推导求“个数”
将等式 $\gcd(n, i) = d$ 两边同除以 $d$，得到等价条件：

$$\gcd\left(\frac{n}{d}, \frac{i}{d}\right) = 1$$

由于 $i$ 的范围是 $1 \le i \le n$，等式两边同除以 $d$ 后得到范围：

$$1 \le \frac{i}{d} \le \frac{n}{d}$$

问题精确转化为：在 $1$ 到 $\frac{n}{d}$ 的正整数中，有多少个数与 $\frac{n}{d}$ 互质？
根据数论定义，这恰好是**欧拉函数 $\phi\left(\frac{n}{d}\right)$** 的值。

### 4. 终极打包公式
满足 $\gcd(n, i) = d$ 的 $i$ 共有 $\phi\left(\frac{n}{d}\right)$ 个，它们对总和的贡献完全一致。因此，原求和公式可等价替换为按 $n$ 的约数 $d$ 进行遍历：

$$\text{Ans} = \frac{1}{n} \sum_{d \mid n} \phi\left(\frac{n}{d}\right) m^d \pmod M$$

### 5. 算法复杂度对比
* **基础版 (暴力遍历)**：需要严格执行 $n$ 次循环。若 $n = 10^6$，需循环 1,000,000 次，复杂度为 $O(N \log N)$。
* **优化版 (枚举约数)**：寻找 $n$ 的约数只需遍历至 $\sqrt{n}$。若 $n = 10^6$，循环次数降至 1,000 次，且 $10^6$ 仅有 240 个实际约数。时间复杂度成功降维至 $O(\sqrt{N})$。

## 三、 优化版核心代码逻辑映射

将终极数学公式映射到代码实现中，即为遍历至 $\sqrt{n}$ 寻找约数，并成对累加欧拉函数贡献的过程：

```cpp
// 寻找 n 的所有约数 d，复杂度 O(sqrt(n))
for (long long i = 1; i * i <= n; i++) {
    if (n % i == 0) {  
        // 1. 找到较小的约数 d1 = i
        long long d1 = i;
        // 累加贡献：个数 phi(n/d1) * 方案数 m^d1
        ans = (ans + get_phi(n / d1) % MOD * power(m, d1)) % MOD;
        
        // 2. 找到对应的较大的约数 d2 = n / i
        long long d2 = n / i;
        // 避免 n 为完全平方数时，中间的约数被重复计算
        if (d1 != d2) { 
            // 累加贡献：个数 phi(n/d2) * 方案数 m^d2
            ans = (ans + get_phi(n / d2) % MOD * power(m, d2)) % MOD;
        }
    }
}
```
