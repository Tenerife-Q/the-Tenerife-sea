# 算法解析：Counting Necklaces (项链计数问题)

这道题是组合数学与群论中极其经典的问题。我们将抛开高深晦涩的群论概念（如群作用、轨道稳定子定理等），将实际问题**抽象为数学定理**，并用最直观的**“画表格算两次”**和**“操场跑圈”**的日常逻辑，为你详细推导并证明最终的计算公式。

---

## 一、 将实际问题抽象为数学问题

**实际问题**：
用 $m$ 种颜色的珠子，串成一条长度为 $n$ 的首尾相连的项链。由于项链是可以**旋转**的（转动后看起来一样的视为同一种），求本质不同的项链总数。结果对 $10^9+7$ 取模（$1 \le n, m \le 10^6$）。

**抽象难点**：
如果是一条固定的直线，排列方案数显然是 $m^n$。但连成项链后，不同的方案在旋转后会发生重复。
且**每种项链重复的形态数量是不一样的**：
- `红-蓝-红-蓝` 转 2 步就和自己重合了，它只有 **2** 个不同形态。
- `红-红-红-红` 怎么转都一样，它只有 **1** 个形态。

正因为每种情况“变出的花样”数量不同，所以**不能简单地用 $m^n$ 除以 $n$**。我们需要引入一个强大的数学工具：**Burnside 引理**。

---

## 二、 定理的本质与两大核心问题

Burnside 引理提供了一种绝妙的逆向思维：**不去数项链，去数“旋转操作”**，看看在每次旋转下，有多少种直线排列方案是“长得一模一样”的，然后求个平均值。

> **核心定理（Burnside 引理大白话版）**：
> 本质不同的项链总数 = 把所有旋转操作下【保持不变的方案数】加起来 $\div$ 旋转操作的总数 $n$。

理解这个定理，只需要彻底搞懂以下**两个核心问题**：
1. **为什么“保持不变的数量求平均值”，刚好就是真实的不同项链数？**
2. **在每次旋转下，怎么计算“保持不变”的数量？**

---

## 三、 核心问题 1：画表格证明“求平均值”的本质

我们用数学中非常强大的思想：**算两次 (Double Counting)** 来证明为什么求平均值是对的。

假设 $n=4$ 颗珠子，我们将所有的直线排列方案和 4 种旋转操作（转 0、1、2、3 步）画成一张**二维表格**。
- **填表规则**：如果某个方案在经历某种旋转后，**长得和原来一模一样（保持不变）**，在格子里填 `1`；如果变了，填 `0`。

我们拿几条具有代表性的项链来举例（A 家族有 4 种形态，B 家族有 2 种形态，C 家族有 1 种形态）：

| 旋转步数 \ 具体的直线方案 | A形1 | A形2 | A形3 | A形4 | B形1 | B形2 | C形1 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **转 0 步 (不转)** | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
| **转 1 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |
| **转 2 步** | 0 | 0 | 0 | 0 | 1 | 1 | 1 |
| **转 3 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |

现在，我们想知道**整个表格里一共有多少个 `1`**。有两种数法：

#### 方法 A：按行数（横向看）
把每一行里的 `1` 加起来，这就等于：**“在每种旋转操作下，保持不变的方案数总和”**。

#### 方法 B：按列数（纵向看，见证奇迹的时刻）
我们把属于**“同一条真实项链（家族）”**的列打包在一起看：
- **项链 A 家族**：有 4 种形态，占了 4 列。因为每种形态只在转 0 步时和自己一样，所以每列只有 1 个 `1`。整个家族贡献的 `1` 数量为：$4 \times 1 = \mathbf{4}$。
- **项链 B 家族**：有 2 种形态，占了 2 列。因为每种形态在转 0 步和转 2 步时和自己一样，所以每列有 2 个 `1`。整个家族贡献的 `1` 数量为：$2 \times 2 = \mathbf{4}$。
- **项链 C 家族**：有 1 种形态，占了 1 列。它在 4 种旋转下都和自己一样，这 1 列有 4 个 `1`。整个家族贡献的 `1` 数量为：$1 \times 4 = \mathbf{4}$。

**极其震撼的结论出现了：**
无论一条项链长什么样，**每一条“本质不同”的独立项链，它整个家族在表格里贡献的 `1` 的总数，永远刚好是 $n$ 个（本例中是 4）！**

既然每一条真实项链都刚好贡献了 $n$ 个 `1`，那么：

$$
\text{表格里 1 的总数} = \text{本质不同的项链总数} \times n
$$

将“按行数”和“按列数”的结果画上等号，并把 $n$ 移项除过去：

$$
\text{本质不同的项链总数} = \frac{\text{每种旋转保持不变的方案数总和}}{n}
$$

**（证明完毕！这就解释了为什么是求平均值）**

## 四、 核心问题 2：操场跑圈证明 $\gcd(n, k)$

现在任务变成了：把一条长度为 $n$ 的项链旋转 $k$ 步，如果它要“长得一模一样”，必须要满足什么条件？
- 答案是：原来在位置 $x$ 的珠子，必须和位置 $(x+k) \bmod n$ 的珠子**颜色相同**。
- 这会导致珠子形成若干个**“必须同色的闭环分组”**。每个分组可以独立从 $m$ 种颜色中任选 1 种。

> **结论：转 $k$ 步产生的分组数量，恰好等于 $\gcd(n, k)$（$n$ 和 $k$ 的最大公约数）。**

### 详细证明过程

假设操场上有 $n$ 个点，你每次跨越 $k$ 个间距。一直跳，直到第一次回到起点，就形成了一个“闭环的分组”。

1. **总共跨越了多远？**
   假设你跳了 $L$ 次刚好**第一次**回到起点，意味着总跨越距离 $L \times k$ 刚好是 $n$ 的倍数。
   因此，总距离必定是 $n$ 和 $k$ 的**最小公倍数**，即 $\text{lcm}(n, k)$。

2. **这个闭环分组里有几个点？**
   跳的次数 $L$，就是这个分组里的点数：

   $$
   L = \frac{\text{lcm}(n, k)}{k}
   $$

   根据公式 $\text{lcm}(n, k) = \frac{n \times k}{\gcd(n, k)}$，代入上式得到：

   $$
   L = \frac{\frac{n \times k}{\gcd(n, k)}}{k} = \frac{n}{\gcd(n, k)}
   $$

3. **一共分成了几个组？**
   既然每个组的大小都一样，那一共有几个组呢？

   $$
   \text{分组数量} = \frac{\text{总珠子数 } n}{\text{每个分组的珠子数 } \frac{n}{\gcd(n, k)}} = \gcd(n, k)
   $$

既然分成了 $\gcd(n, k)$ 个独立的组，每个组内部必须同色（从 $m$ 种颜色中选 1 种），所以转 $k$ 步保持不变的方案数就是：

$$
m^{\gcd(n, k)}
$$

结合表格法的平均值定理，得出基础通解公式：

$$
\text{Ans} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)}
$$

## 五、 结合欧拉函数的终极通解公式

基础公式需要计算 $n$ 次，但题目数据范围 $n \le 10^6$，直接计算必定超时。
我们观察到 $\gcd(n, k)$ 的结果一定是 $n$ 的**约数**。我们可以把 $\gcd$ 结果相同的 $k$ **打包合并计算**。

假设 $d$ 是 $n$ 的一个约数。在 $0 \le k < n$ 范围里，有几个 $k$ 会让 $\gcd(n, k) = d$？
- 条件 $\gcd(n, k) = d$ 等价于 $\gcd\left(\frac{n}{d}, \frac{k}{d}\right) = 1$。
- 换句话说，我们要找的是小于 $\frac{n}{d}$ 并且和 $\frac{n}{d}$ **互质**的正整数个数。
- 这恰好就是**欧拉函数 $\phi\left(\frac{n}{d}\right)$** 的数学定义！

于是，我们将按 $k$ 遍历改为按 $n$ 的约数 $d$ 遍历，得出终极优化公式：

$$
\text{Ans} = \frac{1}{n} \sum_{d \mid n} \phi\left(\frac{n}{d}\right) m^d
$$

在这个公式下，我们只需要遍历 $n$ 的约数（$10^6$ 以内的数最多只有 240 个约数），时间复杂度从 $O(N)$ 暴降到 $O(\sqrt{N})$。

## 六、 最终 C++ 代码实现

*注：题目要求对 $10^9+7$ 取模。根据费马小定理，公式最外层的“除以 $n$”，等价于“乘以 $n$ 的模反元素”，即 $n^{MOD-2} \pmod{MOD}$。*

```cpp
#include <iostream>

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂算法：计算 (base^exp) % MOD
long long power(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

// 求正整数 x 的欧拉函数 Φ(x)
long long get_phi(long long x) {
    long long res = x;
    for (long long i = 2; i * i <= x; i++) {
        if (x % i == 0) {
            // 将质因子 i 除干净
            while (x % i == 0) x /= i;
            // 欧拉函数公式推导：res = res * (1 - 1/i)
            res -= res / i;
        }
    }
    // 如果最后剩余一个大于1的质因子
    if (x > 1) res -= res / x;
    return res;
}

int main() {
    // 优化标准输入输出流，加快执行速度
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n, m;
    if (!(cin >> n >> m)) return 0;

    long long ans = 0;
    
    // 核心：枚举 n 的所有约数 d (只需枚举到 sqrt(n))
    for (long long i = 1; i * i <= n; i++) {
        if (n % i == 0) {
            long long d1 = i;
            // 累加 d1 对应的部分：Φ(n/d1) * m^d1
            ans = (ans + get_phi(n / d1) % MOD * power(m, d1)) % MOD;
            
            long long d2 = n / i;
            // 如果 d1 和 d2 不相等（避免完全平方数时重复计算）
            if (d1 != d2) {
                // 累加 d2 对应的部分：Φ(n/d2) * m^d2
                ans = (ans + get_phi(n / d2) % MOD * power(m, d2)) % MOD;
            }
        }
    }

    // 最后除以 n，等价于乘以 n 的逆元 (n^(MOD-2) % MOD)
    ans = (ans * power(n, MOD - 2)) % MOD;
    
    cout << ans << "\n";
    
    return 0;
}
