# 算法解析：Counting Necklaces (项链计数问题)

这道题是组合数学与群论中极其经典的问题。我们将把实际问题抽象为数学定理，并用最直观的**“画表格算两次”**和**“操场跑圈”**的日常逻辑，为你详细推导并证明最终的计算公式。

## 一、 将实际问题抽象为数学问题

**实际问题**：
用 $m$ 种颜色的珠子，串成一条长度为 $n$ 的首尾相连的项链。由于项链是可以**旋转**的（转动后看起来一样的视为同一种），求本质不同的项链总数。结果对 $10^9+7$ 取模（$1 \le n, m \le 10^6$）。

**抽象难点**：
如果是一条固定的直线，排列方案数显然是 $m^n$。但连成项链后，不同的方案在旋转后会发生重复。
且**每种项链重复的形态数量是不一样的**：
- `红-蓝-红-蓝` 转 2 步就和自己重合了，它只有 2 个不同形态。
- `红-红-红-红` 怎么转都一样，它只有 1 个形态。

正因为每种情况“变出的花样”数量不同，所以**不能简单地用 $m^n$ 除以 $n$**。我们需要引入群论中的强大工具：**Burnside 引理（Burnside's Lemma）**。

---

## 二、 定理的本质与两大核心问题

Burnside 引理提供了一种绝妙的逆向思维：**不去数项链，去数“旋转操作”**。我们观察在每次旋转操作下，有多少种直线排列方案是“长得一模一样”（即在该操作下保持不变）的，然后求个平均值。


> **核心定理（Burnside 引理通俗版）**：
> 本质不同的项链总数 = 把所有旋转操作下【保持不变的方案数】加起来 $\div$ 旋转操作的总数 $n$。

理解这个定理，只需要彻底搞懂以下两个核心问题：
1. **为什么对“保持不变的数量”求平均值，刚好就是真实的不同项链数？**
2. **在每次旋转下，怎么计算“保持不变”的数量？**

---

## 三、 核心问题 1：画表格证明“求平均值”的本质

我们用数学中非常强大的思想：**算两次 (Double Counting)** 来证明为什么求平均值是对的。这其实是群论中**轨道-稳定子定理（Orbit-Stabilizer Theorem）**的具象化表达。

假设 $n=4$ 颗珠子，我们将所有的直线排列方案和 4 种旋转操作（转 0、1、2、3 步）画成一张二维表格。
- **填表规则**：如果某个直线方案在经历某种旋转后，**长得和原来一模一样**，在格子里填 `1`；如果变了，填 `0`。

我们拿几条代表性的项链举例：
- **A 家族**：有 4 种形态，每种形态只在转 0 步时不变。
- **B 家族**：有 2 种形态，每种形态在转 0 步和转 2 步时不变。
- **C 家族**：有 1 种形态（全同色），在 4 种旋转下都不变。

| 旋转步数 \ 直线方案 | A形1 | A形2 | A形3 | A形4 | B形1 | B形2 | C形1 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **转 0 步 (不转)** | 1 | 1 | 1 | 1 | 1 | 1 | 1 |
| **转 1 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |
| **转 2 步** | 0 | 0 | 0 | 0 | 1 | 1 | 1 |
| **转 3 步** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |

现在，我们想知道**整个表格里一共有多少个 `1`**。有两种数法：

### 方法 A：按行数（横向看）
把每一行里的 `1` 加起来，这就等于：**“在每种旋转操作下，保持不变的方案数总和”**。

### 方法 B：按列数（纵向看，本质规律的展现）
我们把属于**“同一条真实项链（同一个轨道 Orbit）”**的列打包在一起看：
- **项链 A 家族**：有 4 种形态（轨道大小为 4）。因为每种形态只在转 0 步时和自己一样（稳定子大小为 1），整个家族贡献的 `1` 数量为：$4 \times 1 = \mathbf{4}$。
- **项链 B 家族**：有 2 种形态（轨道大小为 2）。因为每种形态在转 0 步和转 2 步时和自己一样（稳定子大小为 2），整个家族贡献的 `1` 数量为：$2 \times 2 = \mathbf{4}$。
- **项链 C 家族**：有 1 种形态（轨道大小为 1）。在 4 种旋转下都和自己一样（稳定子大小为 4），整个家族贡献的 `1` 数量为：$1 \times 4 = \mathbf{4}$。

**震撼的结论：**
无论一条项链长什么样，**每一条“本质不同”的独立项链，它整个家族在表格里贡献的 `1` 的总数，永远刚好是 $n$ 个！**（因为 `轨道大小 × 稳定子大小 = 群的大小 n`，这就是 Orbit-Stabilizer Theorem 的精髓）。

既然每一条真实项链都刚好贡献了 $n$ 个 `1`，那么：

$$
\text{表格里 1 的总数} = \text{本质不同的项链总数} \times n
$$

将“按行数”和“按列数”的结果画上等号，并把 $n$ 移项除过去：

$$
\text{本质不同的项链总数} = \frac{\text{每种旋转保持不变的方案数总和}}{n}
$$

**这就解释了为什么是求平均值。**

## 四、 核心问题 2：操场跑圈证明 $\gcd(n, k)$

现在任务变成了：把一条长度为 $n$ 的项链旋转 $k$ 步，如果它要“长得一模一样”，必须要满足什么条件？

答案是：原来在位置 $x$ 的珠子，必须和位置 $(x+k) \pmod n$ 的珠子**颜色相同**。
这会导致珠子形成若干个**“必须同色的闭环分组（循环节 Cycle）”**。每个分组可以独立从 $m$ 种颜色中任选 1 种。


> **结论：转 $k$ 步产生的分组数量，恰好等于 $\gcd(n, k)$（最大公约数）。**

### 详细推导（循环群生成元视角）

假设操场上有 $n$ 个点，编号 $0$ 到 $n-1$，你每次跨越 $k$ 个间距。一直跳，直到第一次回到起点，就形成了一个“闭环的分组”。

1. **总共跨越了多远？**
   假设你跳了 $L$ 次刚好**第一次**回到起点，意味着总跨越距离 $L \times k$ 刚好是 $n$ 的倍数。
   因此，这个最短的总距离必定是 $n$ 和 $k$ 的**最小公倍数** $\operatorname{lcm}(n, k)$。

2. **这个闭环分组里有几个点？**
   跳的次数 $L$，就是这个分组里的点数（即轨道长度）：
   $$
   L = \frac{\operatorname{lcm}(n, k)}{k}
   $$
   根据数学公式 $\operatorname{lcm}(n, k) = \frac{n \times k}{\gcd(n, k)}$，代入上式得到：
   $$
   L = \frac{\frac{n \times k}{\gcd(n, k)}}{k} = \frac{n}{\gcd(n, k)}
   $$

3. **一共分成了几个组（循环节数量）？**
   总共有 $n$ 颗珠子，每个组内部有 $L$ 颗珠子，那么组数就是：
   $$
   \text{循环节数量} = \frac{n}{L} = \frac{n}{\frac{n}{\gcd(n, k)}} = \gcd(n, k)
   $$

既然分成了 $\gcd(n, k)$ 个独立的组，每个组内部必须同色（从 $m$ 种颜色中任选 1 种），所以转 $k$ 步保持不变的方案数就是：

$$
m^{\gcd(n, k)}
$$

结合表格法的平均值定理，得出基础通解公式（Pólya 计数定理的特例）：

$$
\text{Ans} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)}
$$

## 五、 结合欧拉函数的终极数论优化

基础公式需要计算 $n$ 次，但题目数据范围 $n \le 10^6$，如果每次都计算 $\gcd(n, k)$ 必定超时（复杂度 $O(N \log N)$）。
我们可以引入数论进行优化。观察到 $\gcd(n, k)$ 的结果一定是 $n$ 的**约数**。我们可以把 $\gcd$ 结果相同的 $k$ **打包合并计算**。

假设 $d$ 是 $n$ 的一个约数。在 $0 \le k < n$ 范围里，有几个 $k$ 会让 $\gcd(n, k) = d$？

条件 $\gcd(n, k) = d$ 等价于：
$$
\gcd\left(\frac{n}{d}, \frac{k}{d}\right) = 1
$$
换句话说，我们要找的是满足 $0 \le \frac{k}{d} < \frac{n}{d}$ 并且和 $\frac{n}{d}$ **互质**的整数 $\frac{k}{d}$ 的个数。
这恰好就是**欧拉函数 $\phi\left(\frac{n}{d}\right)$** 的数学定义！

于是，我们将原来按 $k$ 遍历，改为按 $n$ 的约数 $d$ 进行枚举遍历，得出终极优化公式：

$$
\text{Ans} = \frac{1}{n} \sum_{d \mid n} \phi\left(\frac{n}{d}\right) m^d
$$

在这个公式下，我们只需要遍历 $n$ 的约数（$10^6$ 以内的数最多只有 240 个约数），时间复杂度从线性的 $O(N)$ 暴降到了 $O(\sqrt{N})$，算法性能得到极大飞跃。

## 六、 最终 C++ 代码实现

*注：题目要求对 $10^9+7$ 取模。根据费马小定理（Fermat's Little Theorem），在模一个素数 $p$ 的意义下，除以 $n$ 等价于乘以 $n$ 的模反元素（乘法逆元），即 $n^{p-2} \pmod p$。在我们的代码中，由 `power(n, MOD - 2)` 实现。*

```cpp
#include <iostream>

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂算法：计算 (base^exp) % MOD，复杂度 O(log exp)
long long power(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

// 求正整数 x 的欧拉函数 Φ(x) 
// 即计算小于等于 x 且与 x 互质的正整数个数，复杂度 O(sqrt(x))
long long get_phi(long long x) {
    long long res = x;
    for (long long i = 2; i * i <= x; i++) {
        if (x % i == 0) {
            // 将质因子 i 除干净
            while (x % i == 0) x /= i;
            // 欧拉函数公式：res = res * (1 - 1/i)
            res -= res / i;
        }
    }
    // 如果最后剩余一个大于 1 的质因子（说明 x 本身包含一个比 sqrt(x) 大的素因子）
    if (x > 1) res -= res / x;
    return res;
}

int main() {
    // 优化标准输入输出流，加快执行速度，适合 ICPC 竞赛
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n, m;
    // 不断读入直到文件结束
    if (!(cin >> n >> m)) return 0;

    long long ans = 0;
    
    // 核心优化：枚举 n 的所有约数 d (只需枚举到 sqrt(n))
    for (long long i = 1; i * i <= n; i++) {
        if (n % i == 0) {
            // 情况 1: 约数 d1 = i
            long long d1 = i;
            // 累加 d1 对应的部分：Φ(n/d1) * m^d1
            ans = (ans + get_phi(n / d1) % MOD * power(m, d1)) % MOD;
            
            // 情况 2: 约数 d2 = n / i
            long long d2 = n / i;
            // 如果 d1 和 d2 不相等（避免 n 是完全平方数时，中间的约数重复计算）
            if (d1 != d2) {
                // 累加 d2 对应的部分：Φ(n/d2) * m^d2
                ans = (ans + get_phi(n / d2) % MOD * power(m, d2)) % MOD;
            }
        }
    }

    // 最后除以 n，在模 MOD 意义下等价于乘以 n 的乘法逆元 (n^(MOD-2) % MOD)
    ans = (ans * power(n, MOD - 2)) % MOD;
    
    cout << ans << "\n";
    
    return 0;
}
```

### 以下解释更好理解



好的，我们完全抛弃所有比喻，只盯着**项链**和**珠子**本身，用最严谨且直白的逻辑把这件事说清楚。

### 1. 什么是“一条真正的项链”？
在物理世界上，一条项链是一个**圆环**。
但在计算时，我们只能先算**“直线的颜色序列”**（一共有 $m^n$ 种）。

所谓“一条真正的项链”，指的是：**可以通过旋转互相转换的若干个“直线序列”的集合**。

举个例子（假设 4 颗珠子，2 种颜色）：
*   **第 1 条项链**：包含 `红蓝绿黄`、`黄红蓝绿`、`绿黄红蓝`、`蓝绿黄红` 这 **4** 个直线序列。你在桌子上转动这条项链，刚好能看到这 4 种顺序。
*   **第 2 条项链**：包含 `红蓝红蓝`、`蓝红蓝红` 这 **2** 个直线序列。无论你怎么转，这条项链只能呈现这 2 种顺序。
*   **第 3 条项链**：包含 `红红红红` 这 **1** 个直线序列。转与不转，顺序都不变。

**题目的目标**：就是求出一共有多少条这样的“项链”（即一共有多少个这样的集合）。

因为每个集合包含的直线序列数量不一样（有的包含 4 个，有的包含 2 个，有的包含 1 个），所以你不能直接用总序列数 $m^n$ 去除以 4。

---

### 2. 为什么“求保持不变的序列总数 ÷ n”就是项链的总数？

既然不能直接除，我们换一种数法。
我们让所有的 $m^n$ 个直线序列，都去经历 4 种旋转（转 0 步、1 步、2 步、3 步）。
我们只统计一件事：**旋转后，颜色顺序和原来完全一样的次数。**

我们来看看每一条具体的项链，在这个统计中会贡献多少次“不变”：

*   **看第 1 条项链（包含 4 个序列）：**
    因为它有 4 个不同的形态，每转 1 步就会变成另一个形态。所以这 4 个序列，**都只在“转 0 步”时和自己一样**。
    这 4 个序列各自贡献了 1 次不变，这条项链总共贡献了：$4 \times 1 = \mathbf{4}$ 次不变。

*   **看第 2 条项链（包含 2 个序列）：**
    因为它只有 2 个形态，所以转 0 步和转 2 步时，形态和原来一样。也就是说，这 2 个序列，**各自能在 2 种旋转下保持不变**。
    这 2 个序列各自贡献了 2 次不变，这条项链总共贡献了：$2 \times 2 = \mathbf{4}$ 次不变。

*   **看第 3 条项链（包含 1 个序列）：**
    因为它只有 1 个形态，无论转 0、1、2、3 步，形态都不变。这 1 个序列，**在 4 种旋转下都保持不变**。
    这 1 个序列贡献了 4 次不变，这条项链总共贡献了：$1 \times 4 = \mathbf{4}$ 次不变。

**数学定理的本质就在这里揭开了：**
无论一条项链包含几个不同的直线序列，**属于同一条项链的所有序列，在所有 $n$ 种旋转中，保持不变的次数加起来，永远精确地等于 $n$。**

既然**每一条**项链都恰好贡献了 $n$ 次“不变”。
那么，我们只要算出整个过程中**“总共发生了多少次不变”**，然后**除以 $n$**，得出的结果自然就是**项链的总条数**。

---

### 3. 旋转 k 步时，怎么算有多少个序列“保持不变”？

现在，我们只需要计算：在 $n$ 颗珠子的直线序列中，如果整个序列向右循环移动 $k$ 个位置，有多少种颜色涂法能让序列看起来没变？

要想序列移动 $k$ 步后长得一样，**位置 $x$ 的珠子，必须和位置 $(x+k) \pmod n$ 的珠子颜色完全相同。**

这会导致珠子之间产生“颜色绑定”，形成若干个必须同色的组：
*   $x$ 的颜色 = $x+k$ 的颜色 = $x+2k$ 的颜色 = $\dots$
*   这个绑定链条什么时候会回到起点 $x$ 形成闭环？
*   当移动的总步数是 $n$ 的倍数时。因为每次移动 $k$ 步，所以移动的总步数既是 $k$ 的倍数，也是 $n$ 的倍数。
*   第一次回到起点时，移动的总步数就是 $n$ 和 $k$ 的**最小公倍数 $\text{lcm}(n, k)$**。

**一个同色组里有几颗珠子？**
每移动 $k$ 步包含 1 颗珠子，所以组内的珠子数为：
$$ \frac{\text{lcm}(n, k)}{k} $$
根据数学公式 $\text{lcm}(n,k) = \frac{n \times k}{\gcd(n,k)}$，代入进去，组内的珠子数就是：
$$ \frac{n}{\gcd(n, k)} $$

**一共有几个同色组？**
项链上一共有 $n$ 颗珠子，每个同色组有 $\frac{n}{\gcd(n, k)}$ 颗珠子。
所以同色组的数量就是：
$$ \frac{n}{\frac{n}{\gcd(n, k)}} = \gcd(n, k) $$

**结论：**
旋转 $k$ 步时，项链上的 $n$ 颗珠子被严格划分成了 **$\gcd(n, k)$ 个独立的组**。
同一个组内的珠子必须颜色一样，不同组的颜色互不影响。
既然有 $m$ 种颜色可以选，那么旋转 $k$ 步能保持不变的涂色方案数就是：
$$ m^{\gcd(n, k)} $$

---

### 4. 总结

1. **算单次旋转：** 旋转 $k$ 步时，保持不变的序列数量是 $m^{\gcd(n, k)}$。
2. **算总数：** 把旋转 0 步、1 步... 直到 $n-1$ 步的所有不变数量全加起来。
3. **求项链数：** 根据第 2 节的推导，把这个总和除以 $n$，就是本质不同的项链数。

这就构成了最终的基础公式：
$$ \text{项链总数} = \frac{1}{n} \sum_{k=0}^{n-1} m^{\gcd(n, k)} $$

这就是仅仅针对项链、序列、旋转和颜色本身，没有任何比喻的纯粹数学推导。
