# 深入解析：Bracket Sequences II —— 镜像反射与广义选票定理

本题是经典卡特兰数问题（Bracket Sequences I）的进阶版。当序列的**部分前缀已经给定**时，我们需要求出将序列补充完整的合法方案数。

要彻底吃透这道题，我们需要结合**“数形结合（几何路径反射）”**与**“字符串前缀翻转”**的视角，最终推导出极其优美的**广义选票定理（Ballot Theorem）**公式。

---

## 一、 问题分析与前缀检查

在补充任何括号之前，必须先对题目给出的前缀进行“体检”。设总长度为 $n$，前缀长度为 $k$。
我们遍历给定前缀，统计已有的左括号数 `open` 和右括号数 `close`。

**必死（直接输出 0）的条件：**
1. **奇偶性不符**：总长度 $n$ 不是偶数。
2. **中途透支**：在遍历前缀的任意时刻，如果 `close > open`，说明右括号已经比左括号多，后续无论怎么补都无法挽回。
3. **总量超标**：合法的最终序列必定有 $n/2$ 个左括号和 $n/2$ 个右括号。如果 `open > n/2` 或 `close > n/2`，说明某一种括号已经超过了最终容量。

如果顺利通过体检，我们就可以明确**还需要补充的数量**：
- 还需要加的**左括号数量**： $L = n/2 - \text{open}$
- 还需要加的**右括号数量**： $R = n/2 - \text{close}$

---

## 二、 数学原理解释：为什么要“反射/反转”？

现在的任务是：用 $L$ 个左括号和 $R$ 个右括号，组成一个合法的剩余序列。
如果没有任何限制，在 $L+R$ 个位置里挑 $L$ 个放左括号，总方案数就是 $\binom{L+R}{L}$。
但怎么剔除掉那些**不合法**的方案呢？这里用到了著名的**双射原理（Bijection）**。

### 视角 1：几何网格的镜像反射（André's Reflection Principle）
把补充括号的过程看作在二维坐标系中行走。向右走代表加左括号（$X+1$），向上走代表加右括号（$Y+1$）。
- **起点**：并不是原点，而是带有初始本钱 $m = \text{open} - \text{close}$。（注意 $m \ge 0$）
- **终点**：$(L, R)$。
- **死亡高压线**：新加的右括号绝对不能把本钱 $m$ 亏光。即 $Y$ 不能超过 $X + m$。合法的路径绝不能触碰直线 $y = x + m + 1$。

**反转魔法**：对于任何一条触碰高压线的“死路”，我们从触碰点开始，把后续的路径沿高压线作**镜像对称翻转**。
翻转后，终点 $(L, R)$ 变成了全新的终点 $(L-1, R+1)$。
**结论**：所有不合法的路径，都与走到新终点 $(L-1, R+1)$ 的路径**一一对应**。不合法方案数恰好为 $\binom{L+R}{L-1}$。

### 视角 2：官方标答的字符串反转
官方题解是在字符串上操作：如果后续填入的过程中，某个最短前缀让“右括号比左括号多出了 $m+1$ 个”，此时序列死刑。
将这个最短前缀里的**所有括号互换**（左变右，右变左）。
反转后，原本多出的右括号变成了多出的左括号。经过数量核算，整个剩余序列的左括号总数刚好变成了 $L-1$，右括号变成了 $R+1$。
**结论等效**：不合法序列的数量等同于挑 $L-1$ 个位置放左括号，同样是 $\binom{L+R}{L-1}$。

---

## 三、 公式推导：从减法到极简乘法（广义选票定理）

根据上述原理，合法方案数 = 总方案数 - 不合法方案数：

$$
\text{Ans} = \binom{L+R}{L} - \binom{L+R}{L-1}
$$

**官方标答的公式解析**：
官方标答给出的公式是 $\binom{n-k}{l} - \binom{n-k}{l+d+1}$。
其实它和我们的公式是完全一样的！在官方定义中，$l = L$，$n-k = L+R$，深度 $d = \text{open} - \text{close} = R - L$。
代入角标：$l + d + 1 = L + (R-L) + 1 = R + 1$。
根据对称性 $\binom{N}{K} = \binom{N}{N-K}$，官方减去的 $\binom{L+R}{R+1}$ 恰好就等于 $\binom{L+R}{L-1}$。**两者在数学上完美统一！**

**代码优化的神来之笔（化简减法）**：
在模 $10^9+7$ 的意义下，做减法非常容易产生负数（官方标答不得不使用 `+ M % M` 来防范）。
为了让代码更优雅，我们对公式进行代数化简：

$$
\begin{aligned}
\text{Ans} &= \frac{(L+R)!}{L! \cdot R!} - \frac{(L+R)!}{(L-1)! \cdot (R+1)!} \\
&= \frac{(L+R)!}{L! \cdot R!} \left( 1 - \frac{L}{R+1} \right) \\
&= \frac{(L+R)!}{L! \cdot R!} \cdot \left( \frac{R - L + 1}{R+1} \right) \\
&= \frac{(L+R)! \cdot (R - L + 1)}{L! \cdot (R+1)!}
\end{aligned}
$$

这个化简后的公式就是组合数学中的 **Bertrand 选票定理（Ballot Theorem）**。它通篇只有乘除法，既避开了负数取模的陷阱，又大幅提升了代码的可读性！

## 四、 重点问题回答 (Q&A)

**Q1：代码中为什么要写 `1LL` 或 `2LL`？**
**答**：在 C++ 中，`int` 最大只能存储约 $2 \times 10^9$。本题对 $10^9+7$ 取模，中间两次取模结果相乘（如 $A \times B$）可能达到 $10^{18}$，会引发严重的整型溢出。加上 `1LL *` 是为了强制将计算过程提升到 64 位的 `long long` 空间中，这是一种极其重要的“防御性编程”习惯。

**Q2：如果题目不给任何前缀，这个公式还成立吗？**
**答**：完美成立！如果不给前缀，则 $L = n/2$，$R = n/2$。
代入化简后的公式，分子后缀 $(R - L + 1) = 1$。
分母变成 $(n/2)! \cdot (n/2 + 1)!$。
整体变成了 $\frac{1}{n/2+1} \binom{n}{n/2}$，**它自动退化成了最经典的卡特兰数公式！** 这个公式是兼容一切情况的“终极形态”。

## 五、 综合最优 C++ 代码实现

以下代码基于我们推导出的**广义选票定理乘法公式**编写。逻辑清晰，规避了减法取模，可读性远超官方标答。

```cpp
#include <iostream>
#include <string>
#include <vector>

using namespace std;
using ll = long long;

const int MOD = 1000000007;

// 快速幂算法：用于计算 (base^exp) % MOD
ll power(ll base, ll exp) {
    ll res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = res * base % MOD;
        base = base * base % MOD;
        exp /= 2;
    }
    return res;
}

// 费马小定理求逆元：在模运算下，除以一个数等价于乘以它的逆元
ll modInverse(ll n) {
    return power(n, MOD - 2);
}

int main() {
    // 优化标准 I/O 速度，算法竞赛标配
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;
    
    string prefix;
    cin >> prefix;

    // 1. 奇数长度不可能合法
    if (n % 2 != 0) {
        cout << 0 << "\n";
        return 0;
    }

    // 2. 验证前缀合法性，并统计现有括号数量
    int open = 0, close = 0;
    for (char c : prefix) {
        if (c == '(') open++;
        else close++;
        
        // 任意时刻右括号的数量绝不能超过左括号
        if (close > open) {
            cout << 0 << "\n";
            return 0;
        }
    }

    // 3. 最终合法的序列中，左右括号总数必定各为 n/2。不能透支
    if (open > n / 2 || close > n / 2) {
        cout << 0 << "\n";
        return 0;
    }

    // 4. 计算剩余还需要添加的左括号 L 和右括号 R
    int L = n / 2 - open;
    int R = n / 2 - close;

    // 5. 预处理阶乘数组
    // 我们最多需要计算到 (L+R)!，最大可能为 n!，为防越界开到 n + 2
    vector<ll> fac(n + 2, 1);
    for (int i = 1; i <= n; i++) {
        fac[i] = fac[i - 1] * i % MOD;
    }

    // 6. 应用化简后的广义选票定理公式: 
    // 方案数 Ans = (L+R)! * (R - L + 1) / ( L! * (R+1)! )
    
    ll total_len = L + R;
    
    // 计算分子: (L+R)! * (R - L + 1)
    // 注意加入 1LL 防止整数乘法溢出
    ll numerator = fac[total_len] * (1LL * (R - L + 1)) % MOD;
    
    // 计算分母: L! * (R+1)!
    ll denominator = fac[L] * fac[R + 1] % MOD;

    // 计算最终结果: 分子 * 分母的逆元 % MOD
    ll ans = numerator * modInverse(denominator) % MOD;

    // 输出最终合法序列方案数
    cout << ans << "\n";

    return 0;
}
```
