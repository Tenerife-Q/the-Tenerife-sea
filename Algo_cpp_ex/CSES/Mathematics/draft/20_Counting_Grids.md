# 算法解析：Counting Grids (网格计数问题)

这道题是 Burnside 引理在二维平面上的经典应用。我们将延用上一题纯粹的数学逻辑，直接分析网格在不同旋转下的“不变性”，并给出严谨的推导与实现。

---

## 一、 将实际问题抽象为数学问题

**实际问题**：
有一个 $n \times n$ 的正方形网格，每个格子可以涂成黑色或白色（共 $m=2$ 种颜色）。如果两个网格可以通过**旋转**看起来完全一样，则视为同一种网格。求本质不同的网格总数，结果对 $10^9+7$ 取模（$n \le 10^9$）。

**抽象分析**：
- 直线/固定序列的总方案数：一共 $n^2$ 个格子，每个格子 2 种颜色，如果没有旋转，方案数为 $2^{n^2}$。
- 旋转操作：由于网格是一个正方形，它只有 **4 种旋转方式**：
  1. 旋转 $0^\circ$（不转）
  2. 旋转 $90^\circ$
  3. 旋转 $180^\circ$
  4. 旋转 $270^\circ$

根据我们上一题推导出的**核心定理（Burnside 引理）**：
> **本质不同的网格总数 = 在这 4 种旋转操作下【保持不变的涂色方案数总和】 $\div 4$**

现在，我们只需要像上一题那样，分别算出这 4 种旋转下，“保持不变的数量”是多少。

---

## 二、 严格推导：计算 4 种旋转下的“保持不变数”

要想在某种旋转后“看起来没变”，意味着**旋转前在这个位置的格子，必须和旋转后转到这个位置的格子，颜色完全相同**。
这会导致格子被强行绑定成若干个“必须同色的组”。我们只需要数清楚：**在每种旋转下，被分成了多少个组**？

假设格子的坐标是 $(x, y)$，取值范围是 $0$ 到 $n-1$。

### 1. 旋转 $0^\circ$（不转）
- **坐标变化**：$(x, y) \to (x, y)$。
- **分组情况**：没有任何格子发生移动。所有的格子各自独立，不需要和别人同色。
- **组数**：一共 $n^2$ 个格子，就有 $n^2$ 个组。
- **保持不变的方案数**：
  
  $$
  2^{n^2}
  $$

### 2. 旋转 $180^\circ$
- **坐标变化**：$(x, y) \to (n-1-x, n-1-y)$。
- **分组情况**：格子 $(x, y)$ 转半圈后和对角线对称的格子互换了位置。所以每 **2 个格子**必须同色，绑定成一个组。
  - **如果 $n$ 是偶数**：所有格子完美配对，共有 $\frac{n^2}{2}$ 个组。
  - **如果 $n$ 是奇数**：最中心那个唯一的格子 $( \frac{n-1}{2}, \frac{n-1}{2} )$ 怎么转都在原地，自己单独成 1 个组；剩下 $n^2-1$ 个格子两两配对，形成 $\frac{n^2-1}{2}$ 个组。总组数为 $1 + \frac{n^2-1}{2} = \frac{n^2+1}{2}$。
- **数学统合**：在 C++ 中，无论是偶数还是奇数，组数都可以统一用整数除法 $\lfloor \frac{n^2+1}{2} \rfloor$ 表示。
- **保持不变的方案数**：

  $$
  2^{\lfloor \frac{n^2+1}{2} \rfloor}
  $$

### 3. 旋转 $90^\circ$
- **坐标变化**：$(x, y) \to (y, n-1-x)$。
- **分组情况**：一个格子每次转 $90^\circ$，要转 **4 次**才能回到原位。因此，每 **4 个格子**被绑定在一起，必须同色，形成一个组。
  - **如果 $n$ 是偶数**：所有格子完美地以 4 个为一组配对，共有 $\frac{n^2}{4}$ 个组。
  - **如果 $n$ 是奇数**：最中心的格子原地不动（1 个组）；剩下 $n^2-1$ 个格子每 4 个一组。总组数为 $1 + \frac{n^2-1}{4} = \frac{n^2+3}{4}$。
- **数学统合**：同理，在整数除法中，组数都可以统一写为 $\lfloor \frac{n^2+3}{4} \rfloor$。
- **保持不变的方案数**：

  $$
  2^{\lfloor \frac{n^2+3}{4} \rfloor}
  $$

### 4. 旋转 $270^\circ$
- **原理与 $90^\circ$ 完全相同**：每次转 $270^\circ$（即反向转 $90^\circ$），同样是 4 个格子轮回一次。
- **保持不变的方案数**：

  $$
  2^{\lfloor \frac{n^2+3}{4} \rfloor}
  $$

---

## 三、 组装终极公式

将上面 4 种旋转保持不变的方案数全部相加，最后除以旋转总次数 $4$。

$$
\text{Ans} = \frac{1}{4} \left( 2^{n^2} + 2^{\lfloor\frac{n^2+1}{2}\rfloor} + 2 \times 2^{\lfloor\frac{n^2+3}{4}\rfloor} \right) \pmod{10^9+7}
$$

**拿题目的例子验证一下**：
$n=4$：
- $n^2 = 16$
- $0^\circ$ 组数：$16$
- $180^\circ$ 组数：$\lfloor \frac{16+1}{2} \rfloor = 8$
- $90^\circ$ 和 $270^\circ$ 组数：$\lfloor \frac{16+3}{4} \rfloor = 4$

总和 $= 2^{16} + 2^8 + 2 \times 2^4 = 65536 + 256 + 32 = 65824$
答案 $= 65824 \div 4 = \mathbf{16456}$。（完美契合题目的 Example Output ！）

## 四、 C++ 代码实现

这道题的代码非常短。唯一需要注意的工程细节是**数据越界问题**：
$n \le 10^9$，所以 $n^2$ 最大可以达到 $10^{18}$。
C++ 中的 `long long` 类型的最大值是 $9.22 \times 10^{18}$，完全可以装下 $10^{18}$。所以我们计算组数时，**必须全程使用 `long long`**。

并且由于指数高达 $10^{18}$，求 $2^{\text{组数}}$ 时，快速幂函数可以直接跑 60 次循环算出结果，极度高效。

```cpp
#include <iostream>

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂算法：计算 (base^exp) % MOD
long long power(long long base, long long exp) {
    long long res = 1;
    base %= MOD;
    while (exp > 0) {
        if (exp % 2 == 1) res = (res * base) % MOD;
        base = (base * base) % MOD;
        exp /= 2;
    }
    return res;
}

int main() {
    // 优化输入输出流
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    long long n;
    if (!(cin >> n)) return 0;

    // 因为 n 最大是 10^9，n * n 最大是 10^18
    // 必须用 long long，不会溢出（long long 上限是 9.22 * 10^18）
    long long n2 = n * n;

    // 严格按照公式推导，计算出四种旋转下，各自被分成的“组数”（即指数部分）
    long long e0 = n2;
    long long e180 = (n2 + 1) / 2;
    long long e90 = (n2 + 3) / 4;
    long long e270 = e90; // 270度与90度完全对称

    // 将四种情况保持不变的方案数相加，边加边取模
    long long sum = 0;
    
    // 注意：根据费马小定理，指数部分可以对 (MOD - 1) 取模来减小计算量
    // 即使不取模，直接把 10^18 丢进快速幂，由于是二分计算，循环也只要执行 60 次，完全不会超时。
    // 为了严谨，我们直接在指数上 %(MOD - 1)
    sum = (sum + power(2, e0 % (MOD - 1))) % MOD;
    sum = (sum + power(2, e180 % (MOD - 1))) % MOD;
    sum = (sum + power(2, e90 % (MOD - 1))) % MOD;
    sum = (sum + power(2, e270 % (MOD - 1))) % MOD;

    // 最后的总和除以 4，等价于乘以 4 的乘法逆元：power(4, MOD - 2)
    long long ans = (sum * power(4, MOD - 2)) % MOD;

    cout << ans << "\n";

    return 0;
}
